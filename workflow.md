# 论文解读文档写作流程

**产物**：纯 HTML + KaTeX（CDN）+ 图片 base64 内联。单文件可离线打开。

**读者画像**：聪明的外行——有技术基础但不懂这个领域。30 秒能抓要点、3 分钟读摘要、10 分钟读全文。

**模板**：`template/` 目录下三套 HTML 骨架（主文档 / 简要总结 / 常见问答），拷贝改。模板已实现本文件描述的默认结构、样式、折叠块、移动端表格滚动——直接改占位符即可，不要从空白 HTML 写起。

**适用范围**：有明确方法 + 实验的论文。纯理论、纯工程、综述类论文不适合本结构，硬套会变形——先判断论文类型再决定是否用这套流程。

---

# 一、流程

### 1. 收集证据

- 下载论文 PDF；arXiv 论文同时下载 [TeX 源码包](https://arxiv.org/src/{arxiv_id})，解压到 `/tmp/<arxiv_id>/`
- 优先读 TeX 源码，没有则用 `pdftotext` 提取
- 搜 GitHub 找官方源码（论文名/作者/机构），clone 或浏览核心文件
- 收集图片：TeX 源码 `figs/` > GitHub 仓库 > arXiv HTML > PDF 截图（最后手段）
- 每个要写的论断记下出处（§X / Table X / 代码文件:行号）

### 2. 判断论文是否适合本格式

本结构适合"有方法 + 有实验"的论文。检查：
- 论文有明确可命名的方法吗？
- 论文有可量化的实验结果吗？
- 论文的核心贡献能用一句话讲清吗？

三项都"是" → 继续。任何一项"否" → 考虑简化结构或不写。

### 3. 设计文档骨架（动笔前想清楚，不进 HTML）

回答三个问题，答案贯穿全文：

1. **一句话本质**：这篇论文用 X 解决了 Y 问题
2. **三个关键数字**：哪三个数字最能证明"解决了"
3. **一个核心类比**：用日常场景把方法讲清楚，贯穿全文而非只用一次

### 4. 准备目录

```
<method-name>/
├── index.html       主文档
├── summary.html     3 分钟摘要
├── qa.html          常见问答（可选）
└── paper.pdf        论文原文
```

拷贝 `template/` 对应骨架，把 `paper.pdf` 放进去。

### 5. 写主文档 index.html

按"规范"部分（见下文）的结构、笔墨分配、写作要点写。模板已实现默认结构，改占位符即可。

### 6. 写简要总结 summary.html

300-500 字，4 段：问题（日常类比）/ 方案（讲思路不讲细节）/ 效果（1-3 个数字表格）/ 启示（可选）。

内容来源：从主文档顶部 callout + 实验章节提炼。如果主文档已通过证据核查，summary 直接复用其数字，不需要重读论文——但写完要交叉核对数字一致。

### 7. 插图

脚本输出 data URI 字符串，粘进 `<img src="...">` 的引号里：

```bash
python template/img_to_b64.py figs/arch.png
# 输出：data:image/png;base64,iVBORw0KGgo...
```

> **脚本输出已是完整 data URI**（含 `data:image/...;base64,` 前缀）。批量替换时直接用整串，不要再拼前缀——双重前缀会让浏览器无法解析图片。

```html
<img src="data:image/png;base64,iVBORw0KGgo..."
     alt="图 1：模型架构（论文 Figure 1）"
     style="width: 100%; max-width: 700px; display: block; margin: 1em auto;">
```

每张图三段结构：引导句（图要回答什么）→ 图片 → 解释段（图说明了什么）。主文档 3-5 张，覆盖问题/方法/实验至少各一张。

### 8. 核查

按"核查清单"（见下文）逐项实查。**不是跑脚本就算通过**——外行友好度四项必须模拟读者实际读一遍。

# 二、规范

## 主文档结构

严格按此顺序，H1 标题用结论式（"一、问题：X 都背着 Y 包袱"）而非主题式（"一、问题"）：

```
顶部 callout（30 秒层）
  问题 → 方案 → 效果，各一句话。读完能向别人复述。

论文元信息 blockquote
  论文名 / 作者 / 单位 / arXiv 链接 / 代码仓库 / 发布时间，每字段一个 <p>

"这篇论文一句话"段
  一段话，把一句话本质展开成 2-3 句。callout 和正文的过渡。

术语速查表
  只列正文反复出现的术语。每个用日常语言解释（不能用未解释的术语解释术语）。
  只在方法章节出现一次的术语不进表，直接在出现处括号注释。

一、问题（H1，结论式标题）
  本章你将知道：<一句话>
  先讲痛点（日常场景），再引出本文要解决什么。不堆方法细节。

二、方法（H1，结论式标题）
  本章你将知道：<一句话>
  先讲直觉（核心类比 + 一句话本质），再给公式，最后讲性质。
  公式推导按需展开——默认只给最终式，推导放 <details> 折叠。

三、实验（H1，结论式标题）
  本章你将知道：<一句话>
  只突出 3 个关键数字（表格里加粗 + 正文 callout 强调）。
  其余数据进表格供深挖，不在正文逐行复述。

四、总结（H1，结论式标题）
  本章你将知道：<一句话>
  方法本质回顾 + 局限 + 启发，各 1-2 句。

证据边界声明（H1）
  本章你将知道：每个论断的出处，哪些是论文事实，哪些是解读者推断。
  分两段：论文明确结论 + 解读者推断。
```

每个 H1（含证据边界声明）下面都要有"本章你将知道"——既是读者导航，也是写完后自检"这章是否传达了预期信息"的标尺。

## 笔墨分配

写完检查，超标就砍：

| 章节 | 占全文比例 | 超标怎么办 |
|------|----------|----------|
| 问题 | 15-20% | 砍背景细节，只留痛点 |
| 方法 | 25-30% | 推导细节挪到折叠/附录 |
| 实验 | 25-30% | 只突出 3 个数字，其余靠表格 |
| 总结 | 5-10% | 不超过 200 字 |
| 术语 + 元信息 + 证据声明 | 15-20% | — |

方法章节内部：直觉 40% / 公式与推导 30% / 性质与好处 30%。直觉少于 30% 说明还没想清楚怎么对外行讲。

## 写作要点

**证据**：每个论断必须有出处，出处集中在"证据边界声明"H1 下。正文不出现 `（论文 §X）` 标注（图片 alt 除外）。从 TeX 抄公式先展开自定义宏：`grep -nE '\\(newcommand|renewcommand|def)' main.tex` 列出所有宏，手动替换成标准 LaTeX。

**术语**：首次用前去论文 TeX 源码搜原词，搜不到就不用。中文综述的统称不能当论文术语。翻译术语附论文原词。术语表用日常语言解释，不能用未解释的术语解释术语。

**直觉先行**：方法章节先讲直觉和类比，再给公式。公式推导默认折叠或后置。判断标准：把所有公式块删掉，方法章节应该还能让人懂方法在做什么。

**关键假设不藏折叠块**：方法章节如果有"为了得到这个公式，我们假设 X"这类前提，必须在主路径用 callout 明确标注，不能只放在折叠的"深入"里。否则读者会把"假设下的结论"当成"无条件的事实"。

**完整对比**：实验对比展示所有相关基线在同一条件下的完整表现，不只挑对本文有利的。表格里有几行，正文对比就覆盖几行。

**局限不脑补**：总结里的"局限"必须是论文实验明确覆盖的范围限制，或论文附录里观察到的失败案例。不要写"未在 X 场景验证"这种推测——除非论文自己说了。自己的判断单独列在"解读者推断"段下。

**H1 结论式**：标题写成"一、问题：<一句话痛点>"而不是"一、问题"。把所有 H1 串起来读，应该能看出论文的故事线。

**证明逐行验证**：写完任何数学证明，用 LaTeX 渲染检查每一步逻辑，不能凭印象写"显然 X ≤ Y"。常见错误：跳步、未定义变量、充分条件当必要条件。

**写完先删 20%**：通读一遍，砍掉所有"读了不影响理解"的内容。宁可少写不可堆砌。可砍的常见项：理论联系的多个视角、推导的中间步骤、实验的逐行解读、过度的举例。砍掉后某处读不懂了，再加回来。

# 三、核查清单

写完浏览器打开 HTML，逐项实查。**外行友好度四项必须模拟读者实际读一遍**，不能只跑 grep 脚本。

### 外行友好度（最重要）

- [ ] 只读 callout + 各 H1 标题 + "本章你将知道"，能不能向别人复述论文做了什么？
- [ ] 术语表里有没有用未解释的术语解释术语？换成日常语言
- [ ] 方法章节第一段是不是直觉/类比？如果是公式或推导，重写
- [ ] 找一句最技术的话，想象讲给不懂这个领域的朋友听——他能不能懂？

### 事实准确（不可妥协）

- [ ] 每个论断有出处，集中在"证据边界声明"H1 下
- [ ] 实验对比展示所有相关基线，不只挑对本文有利的
- [ ] 局限只写论文确实观察到的，解读者推断单独列段
- [ ] 证据边界声明区分"论文事实"和"解读者推断"
- [ ] 关键假设在主路径 callout 标注，不只藏在折叠块
- [ ] 每个数学证明逐行验证逻辑，无跳步
- [ ] 无 `【未核对】` `【待确认】` `【待补出处】` 临时标记

### 格式

- [ ] H1 都是结论式，串起来能看出论文故事线
- [ ] 每个 H1 下都有"本章你将知道"（含证据边界声明）
- [ ] KaTeX 无红色 ParseError；行内数学符号都用 `$...$` 包裹
- [ ] 从 TeX 抄的公式已展开自定义宏
- [ ] 正文无 `（论文 §X）` 标注（图片 alt 除外）
- [ ] callout 5-10 个，颜色语义统一（蓝=提示/绿=正向/红=警告/黄=注意/紫=重点/灰=备注）
- [ ] 对比数据用表格，关键数字加粗
- [ ] 每章都有视觉锚点（图/表/callout 至少一个），无连续两屏纯文字
- [ ] 移动端 390px 视口下宽表不溢出（用 `.table-scroll` 包裹超过 4 列的表格）
- [ ] `content.count('\uFFFD')` 为 0

### 简要总结

- [ ] 4 段齐全，300-500 字，无公式
- [ ] 数字与主文档一致（从主文档复制，不得改写）
- [ ] 效果对比展示完整基线，不只挑 0% 的基线对比

# 四、铁律（违反=文档不可用）

1. **证据优先**：每个论断必须有出处。找不到出处就删，绝不凭印象/类比/常识脑补。论文原文 > 官方源码 > 官方文档 > 实测（需标注）。

2. **直觉先行**：方法章节先讲直觉和类比，再给公式。公式推导默认折叠。判断标准：把所有公式块删掉，方法章节应该还能让人懂方法在做什么。

3. **不选择性对比**：实验对比展示所有相关基线在同一条件下的完整表现。表格里有几行，正文对比就覆盖几行——不能只挑对本文有利的数字说事。

4. **关键假设不藏折叠块**：方法章节的任何假设必须在主路径 callout 标注。读者把"假设下的结论"当成"无条件的事实"是文档的失败。

5. **局限不脑补**：局限只写论文确实观察到的。自己的判断单独列"解读者推断"段，不混进"局限"。
