    HSD：用分层概率账本提高草稿接受率

    问题：逐 token 验证能无损，但容易“一处失败、后面全扔”；联合验证更能保留长草稿，却无法枚举整棵序列树来构造精确残差分布。
    方案：HSD 只沿当前草稿路径查看一层层可访问分支，用 capped prefix ratio 防止概率盈余重复记账，再从后往前寻找可接受的最长前缀。
    效果：Qwen2.5 主实验相对 Tokenwise 平均提升 6.7% 解码速度；HumanEval 32B 的 Block Efficiency 提升 12.3%。

    论文名：Overcoming Joint Intractability with Lossless Hierarchical Speculative Decoding

    作者：Yuxuan Zhou, Fei Huang, Heng Li, Fengyi Wu, Tianyu Wang, Jianwei Zhang, Junyang Lin, Zhi-Qi Cheng

    单位：Independent Researcher；Qwen Team, Alibaba Inc.；University of Washington

    arXiv 链接：[arXiv:2601.05724]（本文依据 v2，2026-03-02）

    代码仓库：[ZhouYuxuanYX/Hierarchical-Speculative-Decoding]

    论文状态：arXiv preprint，使用 ICLR 2026 模板；仅凭论文源码不能断言已被 ICLR 2026 接收

    这篇论文一句话：HSD 把“需要看完整棵生成树的联合残差结算”，改写成“沿已生成路径逐层结算可访问分支”，目标是在保持目标模型分布的同时接受更长的草稿。

    术语速查：先把五个核心词说清楚

    本章你将知道：后文反复出现的概率、分支、盈余和封顶分别指什么。

    术语大白话解释
    Draft distribution $q$小模型给出的概率分布，便宜、快，负责先写候选 token。
    Target distribution $p$大模型的真实分布，昂贵、慢；“无损”指最终输出仍严格服从它。
    Accessible branch给定一个已访问前缀后，下一个 token 的完整词表分支；一次 target forward 就能看到这一层所有概率。
    Excess / deficient mass$q$ 比 $p$ 多分的概率是“盈余”，少分的概率是“缺口”；拒绝和重采样本质上是在重新搬运这些概率质量。
    Capped prefix ratio $r^*$在此前累计比值创下峰值的位置“清账”，当前层只计算峰值之后的比值，避免同一份概率盈余被多层重复使用。

    一、问题：联合验证更会保留草稿，却付不起整棵树的结算成本

    本章你将知道：为什么逐 token 验证容易浪费草稿，以及联合验证为什么卡在“joint intractability”。

    论文把瓶颈从“草稿写得够不够好”转向“验证器会不会保留草稿”：drafting phase 的持续增强已出现收益递减，而 verifier 若仍按旧规则大量丢弃候选，再好的 draft 也无法充分转化为吞吐。新的规则还不能靠放宽阈值换速度，因为输出必须保持 target distribution fidelity。

    先看标准 speculative decoding：便宜模型写，大模型审

    把 draft model 想成实习生，target model 想成总编。实习生一次写一段，理想情况是总编一次审完、尽量保留；但最终稿必须和“总编亲自逐字写”具有完全相同的概率分布。标准 tokenwise verification 在每个位置独立验收：

    $$h(x_t)=\min\left\{1,\frac{p(x_t)}{q(x_t)}\right\}$$

    草稿 token 被接受的质量是 $\min\{p(x_t),q(x_t)\}$；若拒绝，再从缺口分布采样：

    $$P_{\mathrm{res}}(x_t)=\frac{p(x_t)-\min\{p(x_t),q(x_t)\}}{\sum_{\tilde{x}_t\in\mathcal V}\left[p(\tilde{x}_t)-\min\{p(\tilde{x}_t),q(\tilde{x}_t)\}\right]}$$

    为什么单 token 容易无损：当前词表是一套完整账本。$q$ 多分出去的总概率，恰好等于它相对 $p$ 少分的总概率，所以拒绝释放的质量能够精确填满缺口。

    联合验证的问题不是算一条路径，而是结算所有替代路径

    给定已起草的 $X_{1:\gamma}$，单条路径的联合概率并不难算：它只是条件概率连乘。真正困难的是：一旦整段被拒绝，要构造严格无损的序列级残差分布，就需要对 $\mathcal V^\gamma$ 中所有替代序列归一化。路径数随草稿长度指数增长，这才是论文所说的 joint intractability。

    Tokenwise
    账本完整、容易纠错；但前面某处失败后，后面的候选常常也随之丢弃。

    Joint verification
    可以整体判断并保留更长前缀；但完整序列残差账本不可访问。

    验证器必须同时满足三项约束：输出分布与 target 完全一致；只使用当前 forward 已暴露的信息；尽可能保留更多草稿 token。Tokenwise 满足前两项却容易浪费草稿，直接 joint verification 改善第三项却遇到不可访问的全局残差。

    这张论文主图先只看红色叉号和虚线路径：它们表示验证失败后被拒绝、丢弃的草稿。

    矛盾在于：验证器若想少丢内容，就必须跨 token 联合判断；但它只能访问当前路径上的下一步分支概率，不能枚举所有替代路径。HSD 的设计将在下一章解释。

    二、方法：HSD 沿路径逐层结账，而不是枚举整棵生成树

    本章你将知道：partial recovery、分支层级、capped ratio、反向扫描和一次重采样如何拼成 HSD。

    核心类比：总账不可见，就按部门逐层调账

    整棵序列树像一家公司总账，分支像部门账。HSD 不试图一次拿到总账；它只查看当前前缀下“下一 token 的完整部门账”，计算这一层有多少盈余和缺口。如果本层盈余不够，就把缺口交给更高层、更大的兄弟分支共同承担。

    回看图 1：蓝色区域是保留的 $X_{1:\tau}$。HSD 从 $\gamma$ 向左扫描到 $\tau$，在 $\tau+1$ 只做一次纠错采样；更后的候选被丢弃，下一轮再从新 token 后继续。

    第一步：用 generalized divergence 描述局部缺口

    对完整空间 $\Omega$ 的子集 $\Omega'$，论文定义：

    $$D_{\Omega'}(p,q)=\sum_{\omega\in\Omega'}[p(\omega)-q(\omega)]_+$$

    量含义
    $D_{\Omega'}(p,q)$子集内 target 比 draft 多出的总质量，即待补缺口。
    $D_{\Omega'}(q,p)$子集内 draft 比 target 多出的总质量，即拒绝可释放的盈余。
    $[a]_+$$\max(a,0)$，只保留正差值。

    论文 Theorem 1 声称：当且仅当 $D_{\Omega'}(p,q)\le D_{\Omega'}(q,p)$ 时，子集里的缺口能由本地拒绝质量完全恢复。直觉就是“可动用的钱不少于欠款”。

    第二步：利用 autoregressive model 真正可访问的分支

    对前缀 $X_{1:t-1}$，可访问分支为：

    $$\mathrm{Branch}(X_{1:t-1})=\{(X_{1:t-1},\tilde{x}_t):\tilde{x}_t\in\mathcal V\}$$

    论文 Theorem 2 给出一个关键恒等式：

    $$\Delta_{\mathrm{Branch}}(X_{1:t-1})=p(X_{1:t-1})-q(X_{1:t-1})$$

    也就是说，不必枚举更深路径，只看父前缀的 $p-q$ 就知道这一整层总体是缺钱还是有盈余。若前缀比值 $r(X_{1:t-1})=p/q\le1$，本层可以局部恢复；若大于 1，则需要从更高层分支调入质量。

    第三步：capped prefix ratio 防止重复记账

    令 $m(X_{1:t})$ 是此前前缀联合比值最大的索引；若没有前缀比值超过 1，则令它为 0。论文定义：

    $$m(X_{1:t})=\arg\max_{1\le i<t}r(X_{1:i})$$

    $$r^*(X_{1:t})=\min\{r(X_{1:m(X_{1:t})}),1\}\,r(X_{m(X_{1:t})+1:t})$$

    关键直觉：当历史累计比值已超过 1，$\min\{r,1\}$ 把那段截为 1。此前概率盈余由更高层账本负责，当前层只结算峰值之后的 suffix。没有这个 cap，同一份盈余可能在多个层级反复出现。

    基于 $r^*$，HSD 计算 capped branch divergence；完整草稿用 $h_\gamma=\min\{r^*(X_{1:\gamma}),1\}$，更早层级使用：

    $$h_t=\frac{D^*_{\mathrm{Branch}}(p,q\mid X_{1:t})}{D^*_{\mathrm{Branch}}(q,p\mid X_{1:t})},\qquad t<\gamma$$

    接受前缀后，只需从当前分支做一次纠错：

    $$P^*_{\mathrm{res}}(x_t\mid X_{1:t-1})=\frac{\left[q(X_{1:t})(r^*(X_{1:t})-1)\right]_+}{D^*_{\mathrm{Branch}}(p,q\mid X_{1:t-1})}$$

    第四步：从右往左找最长通过层级

    HSD 对每个层级采样独立的 $\eta_t\sim U(0,1)$，从 $\gamma$ 向 1 扫描，第一个满足 $\eta_t\le h_t$ 的位置就是接受长度：

    $$\tau=\max\left(\{0\}\cup\{i\le\gamma:\eta_i\le h_i\}\right)$$

    因此它不是“从左到右一失败就停”，而是直接问：整段能否接受？不行的话，长度 $\gamma-1$ 能否接受？逐级回退，直到找到最长可保留前缀。

    论文用一个 GSM8K 生成前缀来展示这一过程。

    在该单个示例中，HSD 的层级接受向量为 $\{0.1231,1,1,1,0,0,0,0,0,0\}$，可保留 4 个 token；tokenwise 在第一处拒绝时可能丢弃整段。这只是机制示例，不代表数据集平均表现。

    证明边界：论文声明 HSD provably lossless，但 arXiv v2 附录的关键证明存在未证明 Lemma、索引与端点问题；证明思路和逐式审计移至文末“证据边界声明”。

    论文如何解释“无损”：混合分布边界逐段左移

    附录定义混合分布：

    $$F_l=q(X_{1:m_l^*})\,p(X_{m_l^*+1:\gamma}\mid X_{1:m_l^*})$$

    $F_l$ 的左半段服从 draft，右半段服从 target。论文的证明思路是让第 $l$ 层重采样质量等于 $F_{l-1}-F_l$。把各层相加会望远镜消去：

    $$F_L+\sum_{l=1}^{L}(F_{l-1}-F_l)=F_0=p(X_{1:\gamma})$$

    重要证明审计：论文声称 HSD “provably lossless”，但 v2 TeX 的附录证明按字面存在未证明的关键 Lemma、$F_l$ 推导索引错位、最后 suffix ratio 大于 1 的情形未被有效排除，以及顶层残差项与主文 Eq. (20) 的 $q$ 因子不一致。上面的 telescoping 是论文意图；当前文档不能据此独立确认定理已经被附录严格证明。

    展开：证明文本中的具体问题
    • Appendix C 的 Lemma 5–7 直接陈述 capped ratio 和重采样质量性质，没有给出证明，而 Theorem 6 依赖它们。
    • 定义 $F_l$ 对应 $m_l^*$，但推导 $F_{l-1}$ 时中间使用了 $m_l^*\to m_{l+1}^*$，最后又跳回 $m_{l-1}^*\to m_l^*$。
    • 证明声称最后的 $r(X_{m_L^*+1:\gamma})>1$ 会产生新 capping index；但定义只查看 $t\le\gamma$ 的严格前缀，位置 $\gamma$ 创新高并不会进入该集合。
    • 附录顶层残差写成 $[r^*-1]_+q(X_{1:\gamma-1})p(x_\gamma\mid X_{1:\gamma-1})$，主文 Eq. (20) 直接推出的质量则含 $q(X_{1:\gamma})$。

    这些是对 arXiv v2 源码的逐式审计，不等价于证明算法一定错误；它意味着需要作者的修正版证明或独立形式化验证。
