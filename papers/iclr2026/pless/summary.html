论文原文

p-less Sampling：用一个数同时回答"留多少、留哪些"

给非专业读者的 3 分钟导读。技术细节看 主文档，论文原文看 paper.pdf。

这个领域原来有什么问题

LLM 每生成一个 token 都会吐出覆盖整个词表的概率分布，直接抽样经常抽到离谱的词。主流做法是"截断采样"——先画线砍掉低概率词，剩下的归一化后抽。问题是这条线几乎总由固定超参数决定（top-$p$=0.9、min-$p$=0.1 等），最优值随任务和温度漂移。温度一拉高分布变平，固定阈值要么放进大量低概率词让文本退化，要么所有词都达不到阈值、只能偷偷退回贪心解码。

本文做了什么

论文提出 p-less Sampling：把"两次独立随机抽样命中同一 token 的概率"——分布上每个 token 概率的平方和——当作截断门槛。这个数在信息论里是 Rényi 熵 order 2 的负指数，随分布聚集度自动变化：分布集中时它大、门槛严；均匀时它小、门槛松，但下界卡在均匀分布概率上，不会无差别放低概率词进来。4 行 PyTorch 实现，零超参。变体 p-lessnorm 门槛更宽松，适合创意写作。

效果如何

3 模型 × 4 推理数据集的 AUC：Llama2 和 Mistral 上 8 个单元格全部最高或并列最高；Llama3-70b 上 8 个中 5 个最高、3 个次高（差距 < 0.005）。Llama2-7b AUC 完整对比：

高温段差距更大：τ=2.0 写作胜率 Llama2 p-less 65.64%，top-p/ε/η 归零，最强基线 mirostat 仅 26.88%、min-p 48.94%；Mistral p-less 60.32% vs min-p 54.11%。每 token 采样时间比 min-p 快 22%。p-less 并非所有 setting 都赢——τ=1.0 写作 ε-sampling 62.18% 略胜 p-less 55.08%——但核心优势在高温段几乎不退化。

一句话启示

信息论量（Rényi 熵的指数）可以替代人工超参，让采样器自适配任务和温度——把"调超参"这件事交给分布自身。

局限：实验只覆盖 7B-70B 规模、5 个数据集、τ∈[0.5, 2.0]；论文附录观察到的失败模式包括"复杂算术步骤触发熵 spike 仍会算错"（Pattern 1）和"题目表述歧义时仍会误解题意"（Pattern 2）。
