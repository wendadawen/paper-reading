    RLHF：从偏好数据到 ChatGPT

    问题：SFT 只能模仿人写的示范，学不到「人类更偏好什么」；示范标注贵、偏好标注便宜，但偏好是相对的（A 比 B 好），没法直接拿来监督训 policy。
    方案：三步管线——SFT 拿到 $\pi_{\text{ref}}$ → 用偏好数据训一个标量奖励模型 $r_\phi$（把「相对偏好」压成「绝对分数」）→ 用 PPO 最大化 $r_\phi$，同时加 KL 约束锁住 policy 别跑飞。
    产物：看懂 ChatGPT 对齐是怎么做出来的、每一步为什么不可省、KL 约束到底防什么、为什么 DPO 出现后 RLHF 还没死。

    30 秒上下文：先知道它站在哪里

    大流程
    预训练（next-token）→ SFT（指令跟随）→ 对齐（RLHF 在这一段）
    旧方案
    SFT only：人写「指令 → 好答案」示范，模型做交叉熵监督学习
    痛点
    示范贵；表达不了「A 比 B 好」；CE loss 和「人类偏好」之间隔一层
    改变点
    换成偏好数据 → 学标量 reward → 用 RL 最大化 reward（带 KL 约束防 reward hacking）

    读完你能回答

    • RLHF 三步分别是什么？为什么这三步缺一不可？
    • RM 用什么 loss 训？为什么是 Bradley-Terry 模型？
    • PPO 阶段的 reward 为什么要减去 KL 项？不加会怎样？
    • RLHF 和 DPO 解决的是同一个问题，为什么工业界两个都要？
    • RLHF 训练不稳定主要体现在哪里？

    最容易误解

    • 「RLHF = PPO」——错。PPO 只是第 3 步，前两步 SFT 和 RM 是基础，没有 RM 就没有 reward 信号，PPO 无从优化。
    • 「RM 直接给好答案打高分就行」——错。RM 是从「A 比 B 好」的相对比较里学的，不是从绝对分。它学的是一个能让「好答案的分数 > 差答案的分数」的标量函数。
    • 「KL 约束是为了让模型不遗忘预训练知识」——不准确。主要目的是防 reward hacking：policy 钻 RM 的空子拿高分但回答其实很烂。锁在 $\pi_{\text{ref}}$ 附近 = 不让它偏离到 RM 没见过的区。
    • 「有了 DPO 就不用 RLHF 了」——看场景。DPO 不能探索新回答（只用静态偏好数据），RLHF 能持续采样让 RM 打分。两者是互补的。

    来源：InstructGPT（Ouyang et al. 2022）——3 步管线原版；Christiano et al. 2017《Deep RL from Human Preferences》——偏好 + RL 奠基；Bradley & Terry 1952——偏好模型；Schulman et al. 2017——PPO。

    前置：监督学习、交叉熵、softmax、policy gradient 基础。不会的先看 backward。

    一句话：RLHF = 让人标「A 好还是 B 好」→ 训一个打分模型（RM）把这些相对偏好压成绝对分数 → 用 PPO 把对话模型往高分方向推，加 KL 约束防它钻空子跑飞。三步管线把「人类偏好」这个没法直接写 loss 的东西，变成可优化的标量目标。

    一、定位：它在整个流程里的哪一步

    RLHF 是大语言模型训练流程的最后一段——对齐段。整条链是这样的：

    预训练（海量无标注文本，next-token） ↓ SFT（人写示范，指令 → 好答案，仍监督学习） ↓ RLHF（人标偏好 A vs B，训 RM，PPO 优化 policy） ← 本篇 ↓ 对齐后的 chatbot（ChatGPT、Claude 这类）

    上游：SFT 后的模型，记作 $\pi_{\text{ref}}$（reference model），它是 RLHF 的起点和约束锚点。

    下游：一个能在对话里按人类偏好回答的 chatbot。

    RLHF 不是单一算法，是一个管线：SFT → RM → PPO。三步用三种不同的数据和三种不同的 loss。后面每一步都依赖前一步的产物。

    二、旧世界：没有它时会怎么做

    RLHF 出现前，对齐主要靠 SFT only：找一批标注员，针对每个指令写一条高质量回答，组成 $(x, y)$ 数据对，模型做交叉熵监督学习：

    $$\mathcal{L}_{\text{SFT}} = -\sum_t \log \pi(y_t | x, y_{

    这个方案看起来很合理：

    • 标签干净：人写的就是「正确答案」，没有歧义
    • 训练稳：和预训练同款 loss，工程上零额外成本
    • 评估容易：CE loss 直接反映模仿质量

    它依赖两个隐含假设：

    1. 假设 1：人能写出比模型更好的示范。模型弱的时候成立；模型变强后，找这种人的成本爆炸。
    2. 假设 2：一条好答案 = 偏好的完整表达。即「想让模型输出这个」等价于「模型应该这么做」。

    三、断点：为什么旧办法自然走不下去

    SFT 的合理性在模型变强、需求变复杂时一步步瓦解。具体断点落在四个地方：

    断点 1：标注成本

    写一条高质量示范 ≈ 比较两个答案谁更好的 10–100 倍成本。前者要人写一整段，后者只要打个勾。模型变强后，「能写出比模型更好答案」的人越来越少，单条示范的成本指数上升。

    断点 2：信号表达力

    示范只给「一个好答案」，但无法表达：

    • 「A 比 B 好」这种相对判断
    • 「好到什么程度」这种强度信号
    • 「A 在 helpful 上好，但在 harmless 上差」这种多维信号

    偏好对 $(x, y_w, y_l)$ 天然带相对和多维信息——同一条 prompt 配两个答案，标注哪个好。

    断点 3：目标不一致

    SFT 优化的是「模仿示范的字」（token 级 CE loss），但真正想要的是「人类更偏好」。两者之间隔了一层映射：模仿得像 ≠ 偏好度高。比如示范可能用了某个具体说法，但模型换个等价说法其实同样好——SFT 会惩罚它。

    断点 4：扩展性

    HHH（helpful / harmless / honest）这类多维对齐标准，没法用单条示范表达。但可以用一组偏好对表达：「在 harmless 上 A 比 B 好」。需求越多维，SFT 越使不上劲。

    四个断点合起来：示范贵 + 表达不了偏好 + 目标错位 + 多维扩展不动。不是 SFT 做错了，是 SFT 的信号源本身撑不起对齐的需求。需要换一种更便宜、更表达性、更对目标的数据——偏好数据。

    四、新想法：这个概念改变了哪一步

    顺着断点自然往下推：既然偏好数据便宜又表达性强，能不能直接用它训 policy？

    但偏好数据是相对的（A 比 B 好），policy 训练需要绝对的奖励信号（这条回答值多少分）。中间缺一个把「相对」压成「绝对」的东西。奖励模型 $r_\phi(x, y)$ 就是这个中间层——它从偏好对里学一个标量函数，让 $r_\phi(x, y_w) > r_\phi(x, y_l)$。

    有了标量 reward，policy 就能做 RL 了——采样回答、让 RM 打分、往高分方向更新。但这里又冒出一个新问题：policy 容易钻 RM 的空子（reward hacking），找一些让 RM 给高分但人看着很烂的回答。因为 RM 只在偏好数据覆盖的区域准，覆盖不到的地方它乱给分。

    解决办法：加 KL 约束，把 policy 锁在 SFT 模型 $\pi_{\text{ref}}$ 附近。policy 可以朝高分方向走，但不能离 $\pi_{\text{ref}}$ 太远——离得越远，惩罚越大。这样 policy 只能在「RM 见过的、$\pi_{\text{ref}}$ 也认识的」区域里优化。

    三步管线就这样自然落地：

    1. SFT：拿到 $\pi_{\text{ref}}$（起点 + KL 锚点）
    2. RM：从偏好数据学标量 reward（把相对压成绝对）
    3. PPO：用 reward 做 RL，加 KL 约束防 hacking（往高分走但别跑飞）
    • 读到这里，你应该已经能解释：
    • 旧方案卡在哪一步（信号源撑不起对齐需求）
    • 新概念为什么自然会改这一点（换成偏好数据 + 中间加 RM + RL 加约束）
    • 如果不改，代价会在哪里继续变大（标注成本 + 多维标准无法扩展）

    五、正式定义：它到底是什么

    小白版定义：让人标「A 好还是 B 好」，训一个打分模型，再用这个打分模型当奖励，用 RL 把对话模型往高分方向推——加个约束不让它跑飞。

    正式技术定义：RLHF 是三步对齐管线——

    1. SFT：在示范数据上做交叉熵监督，得到参考模型 $\pi_{\text{ref}}$。
    2. RM：在偏好数据 $\{(x, y_w, y_l)\}$ 上，用 Bradley-Terry 损失训奖励模型 $r_\phi(x, y)$：
      $$\mathcal{L}_{\text{RM}} = -\log \sigma\big(r_\phi(x, y_w) - r_\phi(x, y_l)\big)$$
    3. PPO：用 PPO 最大化 KL 惩罚后的 reward：
      $$\max_{\pi_\theta}\; \mathbb{E}_{x \sim \mathcal{D},\, y \sim \pi_\theta}\bigg[ r_\phi(x, y) - \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)} \bigg]$$ 其中 $\beta$ 是 KL 惩罚系数，控制 policy 偏离 $\pi_{\text{ref}}$ 的强度。

    三步对应三种数据（示范 / 偏好 / prompt）、三种 loss（CE / BT / PPO clipped）、三个产物（$\pi_{\text{ref}}$ / $r_\phi$ / $\pi_\theta$）。

    六、机制：它怎么工作

    直觉层：SFT 让模型会说话，RM 让模型知道什么好，PPO 让模型朝着好的方向走但不离 SFT 太远。

    流程层：用一个贯穿全程的「1+1=」最小例子走一遍。词表就 5 个 token：$\{\text{"1"}, \text{"+"}, \text{"="}, \text{"2"}, \text{"3"}\}$，索引 0-4。prompt 是 "1+1="（token 0,1,2），正确答案 "2"（token 3），错误答案 "3"（token 4）。

    Step 1：SFT 拿到 $\pi_{\text{ref}}$

    SFT 就是在示范数据上做交叉熵。假设有一条示范 ("1+1=", "2")，模型学完之后，在 prompt "1+1=" 后输出 "2" 的概率应该高。

    训练前 $\pi_{\text{ref}}$ 在 prompt 后输出下一个 token 的 logits 是 $[0.5, 0.5, 0.5, 0.4, 0.6]$，softmax 后：

    $$\pi_{\text{ref}} = [0.209, 0.209, 0.209, 0.190, 0.255]$$

    训练后 logits 变成 $[0.1, 0.1, 0.1, 2.5, 0.2]$，softmax 后：

    $$\pi_{\text{ref}} = [0.039, 0.039, 0.039, 0.844, 0.043]$$

    现在模型 84.4% 的概率输出 "2"。SFT 完成，$\pi_{\text{ref}}$ 冻结，作为后续 KL 锚点。这一步没有偏好，只有示范。

    Step 2：RM 从偏好数据学标量 reward

    收集偏好数据：$(x = \text{"1+1="},\; y_w = \text{"2"},\; y_l = \text{"3"})$。RM 是一个小网络 $r_\phi(x, y) \to \mathbb{R}$，输入 prompt + 回答，输出一个标量分数。

    训练目标用 Bradley-Terry 损失：

    $$\mathcal{L}_{\text{RM}} = -\log \sigma\big(r_\phi(x, y_w) - r_\phi(x, y_l)\big)$$

    直觉：让 $r_\phi(x, y_w) > r_\phi(x, y_l)$，差距越大 loss 越小。RM 学到的不是「绝对分」，是「能让好答案分数高于差答案」的标量函数。

    展开：Bradley-Terry 损失怎么来的

    Bradley-Terry 模型假设：$y_w$ 比 $y_l$ 好的概率可以写成它们 reward 差的 sigmoid：

    $$P(y_w \succ y_l \mid x) = \sigma\big(r(x, y_w) - r(x, y_l)\big) = \frac{1}{1 + e^{-(r_w - r_l)}}$$

    观测到「$y_w$ 比 $y_l$ 好」这个事实，做最大似然就是让 $P$ 越大越好，等价于最小化 $-\log P$：

    $$\mathcal{L}_{\text{RM}} = -\log \sigma\big(r_\phi(x, y_w) - r_\phi(x, y_l)\big)$$

    为什么用 BT 不用别的：BT 是偏好建模最简单的参数化——只用 reward 差就能表达偏好概率，且 sigmoid 把任意实数压到 $(0,1)$ 概率。它是 RLHF 偏好建模的事实标准（InstructGPT、Llama、Claude 都用）。

    来源：Bradley & Terry 1952；BT 在 RLHF 中的应用 ← Christiano et al. 2017、InstructGPT §3。

    「1+1=」数值演示：RM 初始化时对 ("1+1=", "2") 给 0.3，对 ("1+1=", "3") 给 0.5（一开始乱给）。$r_w - r_l = -0.2$，$\sigma(-0.2) = 0.450$，$L = -\log 0.450 = 0.799$。

    训练几步后，RM 学到 $r_\phi(\text{"1+1=", "2"}) = 1.8$，$r_\phi(\text{"1+1=", "3"}) = -1.2$。$r_w - r_l = 3.0$，$\sigma(3.0) = 0.953$，$L = -\log 0.953 = 0.048$。RM 现在能正确把 "2" 排在 "3" 前面。这一步学的是相对偏好，不是绝对分。

    Step 3：PPO 用 reward 训 policy，加 KL 约束

    这一步把 RM 当环境，policy $\pi_\theta$ 在里面做 RL。流程是：

    1. 从 prompt 分布采 $x$（"1+1="）
    2. 用当前 $\pi_\theta$ 采样回答 $y$（可能采到 "2" 也可能采到 "3"）
    3. 算 reward = $r_\phi(x, y) - \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$（RM 分数 - KL 惩罚）
    4. 用 PPO clipped 目标更新 $\pi_\theta$
    展开：PPO clipped 目标怎么来的

    PPO 的核心是「用 importance sampling 重用旧 policy 采的样本，但 clip 防止更新幅度过大」。

    设比率 $r_t(\theta) = \frac{\pi_\theta(y_t | \cdots)}{\pi_{\theta_{\text{old}}}(y_t | \cdots)}$（新 policy 概率 / 旧 policy 概率），优势 $A_t$ 是「这个回答比平均好多少」。

    普通 policy gradient 目标：$\mathbb{E}[r_t A_t]$，鼓励「优势为正的回答概率上升」。

    PPO 加 clip：

    $$\mathcal{L}^{\text{CLIP}} = \mathbb{E}\big[\min\big(r_t A_t,\; \text{clip}(r_t, 1-\epsilon, 1+\epsilon) A_t\big)\big]$$

    直觉：当 $A > 0$（好回答），鼓励 $r_t$ 上升但不超过 $1+\epsilon$；当 $A < 0$（坏回答），鼓励 $r_t$ 下降但不低于 $1-\epsilon$。clip 防止单步更新太激进把 policy 改坏。

    为什么需要 value model（critic）：算优势 $A_t = R_t - V_\phi(x)$ 需要 baseline $V$，所以 PPO 在 RLHF 里通常要单独训一个 value model。这是 RLHF 工程复杂度的重要来源（也是 GRPO 想省掉的东西）。

    来源：PPO ← Schulman et al. 2017；PPO 在 RLHF 中的应用 ← InstructGPT §3.3。

    「1+1=」数值演示：$\pi_\theta$ 初始 = $\pi_{\text{ref}}$，采到 "2" 和 "3" 的概率分别是 0.844 和 0.043。$\beta = 0.1$。

    • 采到 "2"：$r_\phi = 1.8$，$\log(\pi_\theta/\pi_{\text{ref}}) = 0$（初始相同），reward = $1.8 - 0 = 1.8$。优势为正 → 推 "2" 概率上升。
    • 采到 "3"：$r_\phi = -1.2$，reward = $-1.2 - 0 = -1.2$。优势为负 → 推 "3" 概率下降。

    几轮后 $\pi_\theta$ 在 prompt 后输出 "2" 的概率从 0.844 升到 0.92，输出 "3" 的概率从 0.043 降到 0.01。但因为有 KL 惩罚，$\pi_\theta$ 不会无限制地把 "2" 推到 0.99——偏离 $\pi_{\text{ref}}$ 越远，KL 项越大，最终平衡在一个点。

    关键：KL 约束防的是 reward hacking。如果不加 KL，policy 会找到 RM 给高分的诡异回答（比如 "1+1=2 2 2 2" 可能让 RM 给 5.0 分，因为 RM 没见过这种）。加了 KL，policy 离 $\pi_{\text{ref}}$ 越远惩罚越大，就锁在「RM 见过的、$\pi_{\text{ref}}$ 也认识的」正常区域里。

    展开：RLHF 常见超参表（InstructGPT / 工程实践）

    $\beta$（KL 系数）0.01 – 0.2大 → 锁得紧，policy 改不动；小 → 容易 reward hacking

    七、边界：什么时候用，什么时候不用

    它解决什么：把「人类偏好」这个没法直接写 loss 的目标，通过偏好数据 + RM 转成可优化的标量 reward，再用 RL 优化 policy。能持续采样探索新回答。

    它不解决什么：偏好数据本身的质量问题（标注员偏见、不一致）；RM 在分布外的乱打分（只能靠 KL 部分缓解）；训练不稳定性（PPO 超参敏感）。

    概念解决什么不解决什么什么时候用
    RLHF（本篇）从偏好学 reward，RL 优化 policy，能探索训练不稳，RM 分布外乱给分，工程复杂预算充足、要 SOTA 对齐质量、需要持续探索
    SFT模仿示范，训练稳学不到偏好，示范贵有高质量示范、只要模仿不需要对齐
    DPO跳过 RM 和 PPO，直接在偏好数据上训 policy只用静态数据，不能探索新回答偏好数据充足、追求稳定、不想引入 RL 复杂度
    RLAIF用 AI 替代人打偏好标签AI 偏见会传给模型，质量上限低于人类标注预算紧、规模大、能接受 AI 偏好
    RLVR / GRPO用规则验证器（如答案匹配）当 reward，跳过 RM只对有客观答案的任务（数学、代码）有效数学题、代码题等可验证任务

    工业界现在流行 DPO 做 base，RLHF 做精细调整：先 DPO 跑出基础对齐（便宜稳定），再用 RLHF 持续探索（贵但效果好）。两者不是对立的。
