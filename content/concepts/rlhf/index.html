    RLHF：用人类偏好训 LLM

    问题：SFT 只能教模型模仿标准答案，但很多问题没有标准答案，只有"谁比谁好"
    方案：用人工标注的偏好对训一个奖励模型（RM），再用 RM 当 reward 信号做 RL 训 LLM
    产物：一个能持续对齐人类偏好的 LLM（ChatGPT 走的就是这条路）

    来源：InstructGPT 论文（Ouyang et al. 2022）+ Christiano et al. 2017 + Schulman et al. 2017 (PPO)

    不是某一篇论文：RLHF 是一个完整流程，由多篇论文拼成。本文档讲整体方法。

    前置：SFT（监督微调）、softmax、loss、梯度下降。不会的先看 backward 和 DPO。

    RLHF 一句话：SFT 教模型"该说什么"（模仿标准答案），RLHF 教模型"什么更好"（对齐人类偏好）。做法是先训一个能代替人工打分的 RM，再用 RM 当 reward 信号做 RL，让模型自己摸索出高分的回答方式。

    一、问题：SFT 教不会模型"谁比谁好"

    本章你将知道：为什么 SFT 不够，需要一种新的训练方式。

    SFT（监督微调）的方式是：给模型看 (prompt, 标准答案) 对，让它模仿。比如 "1+1=" → "2"。这教得会算术题，但很多问题没有唯一标准答案。

    比如"怎么学 Python？"——"看官方文档"和"做项目练手"都不错，没法说哪个是"标准答案"。但你能说"看官方文档"比"不知道"好。这是偏好，不是标准答案。

    SFT 只能给标准答案让模型模仿，没法教"在两个都可以的答案里，挑人类更喜欢的那个"。这是 SFT 的根本局限。

    SFT vs RLHF 数据格式对比：

    • SFT 数据：(prompt, 标准答案)——教模仿
    • RLHF 数据：(prompt, 好答案, 差答案)——教偏好

    这是 RLHF 区别于 SFT 的本质。SFT 教"该说什么"，RLHF 教"什么更好"。

    二、方法：训 RM 当 reward 信号，用 RL 训 policy

    本章你将知道：RLHF 的三步流程；为什么 PPO 需要 ref_policy 和 KL 约束；advantage、value、critic、GAE 等概念怎么来的。

    直觉

    RLHF 要解决的核心问题是：人工标注太贵，但 RL 需要大量 reward 信号。每次 policy 生成一个回答都要打分，靠人工跟不上。解决方案是训一个 RM 代替人工——用少量人工偏好数据训 RM，之后 RM 能批量给任意 (prompt, 回答) 打分。

    整个流程分三步：偏好数据 → 训 RM → 用 RL（PPO 算法）训 policy。下面按顺序讲。

    第 1 步：偏好数据

    偏好数据是 RLHF 的燃料。每条数据是一组 (prompt, 好答案, 差答案) 三元组，人工标注"哪个回答更好"：

    数据格式说明：$(x, y_w, y_l)$

    • $x$ = prompt（题目）
    • $y_w$ = chosen（好答案，win）
    • $y_l$ = rejected（差答案，lose）

    数据量不用太大——几万到几十万条就够训一个 RM。但每条都要人看，所以很贵。这是整个 RLHF 最耗人工的环节。

    第 2 步：训 RM

    RM 是一个打分模型：输入 (prompt, 回答)，输出一个标量分数。训练目标是让好答案的分高于差答案的分。

    训练前 RM 什么都不知道，给所有答案打 0 分。训练后学会"好答案分高，差答案分低"：

    prompt回答训练前 RM 分训练后 RM 分
    1+1=20+1.0
    1+1=30-1.0

    关键假设：RM 学的是相对排序，不是绝对分数。

    r("2")=1.0、r("3")=-1.0，和 r("2")=100、r("3")=98 效果一样——都是"2 比 3 高 2 分"。绝对分没意义，差值才有意义。这个假设贯穿整个 RLHF。

    RM 用 Bradley-Terry 模型训练，把"好答案比差答案好"建模成概率：

    展开：RM 的 loss 公式完整推导（5 步）

    目标：训一个 RM，让好答案 $y_w$ 的分高于差答案 $y_l$。

    第 1 步：把"偏好"变成概率

    我们想知道：RM 认为"$y_w$ 比 $y_l$ 好"的概率是多少？用 Bradley-Terry 模型把这个概率写成 RM 分数的函数。

    BT 模型原始形式（把 reward 当"强度"，强度大的赢概率大）：

    $$P(y_w \succ y_l) = \frac{r(y_w)}{r(y_w) + r(y_l)}$$

    问题：这要求 $r > 0$（概率不能为负）。但 RM 输出的分数可以是任意实数。怎么解决？

    第 2 步：指数换元

    令 $r(y) = e^{s(y)}$，其中 $s(y)$ 是任意实数（可以是负的）。指数函数保证 $r > 0$：

    $$P = \frac{e^{s(y_w)}}{e^{s(y_w)} + e^{s(y_l)}}$$

    第 3 步：化成 sigmoid 形式

    上下同除以 $e^{s(y_w)}$：

    $$P = \frac{1}{1 + e^{s(y_l) - s(y_w)}} = \frac{1}{1 + e^{-(s(y_w) - s(y_l))}}$$

    这就是 sigmoid 函数 $\sigma(x) = \frac{1}{1 + e^{-x}}$ 的形式，令 $\Delta = s(y_w) - s(y_l)$：

    $$P(y_w \succ y_l) = \sigma(\Delta) = \sigma(r_w - r_l)$$

    这里 $r_w = s(y_w)$、$r_l = s(y_l)$ 就是 RM 给两个答案打的分。RM 分数的差过 sigmoid = 偏好概率。

    第 4 步：最大似然——让 RM 给观测到的偏好赋高概率

    我们观测到"$y_w$ 比 $y_l$ 好"这个事实。最大似然就是：调整 RM 参数，让它给这个观测赋的概率 $P$ 尽量大。

    最大化 $P$ 等价于最大化 $\log P$（log 是单调递增函数，不改变大小关系）：

    $$\max \log P = \max \log \sigma(r_w - r_l)$$

    第 5 步：取负，变成最小化问题

    习惯上我们最小化 loss，所以加个负号。最小化 loss = 最大化概率：

    $$\mathcal{L}_{RM} = -\log \sigma(r_w - r_l)$$

    这就是 RM 的 loss。当 $r_w \gg r_l$（好答案分远高于差答案）时，$\sigma \to 1$，$\log \sigma \to 0$，loss $\to 0$。当 $r_w = r_l$ 时，$\sigma = 0.5$，loss = $-\log 0.5 = 0.693$（中性状态）。

    关键符号解释：

    • $r_w = r_\varphi(x, y_w)$：RM 给 (prompt, 好答案) 打的分
    • $r_l = r_\varphi(x, y_l)$：RM 给 (prompt, 差答案) 打的分
    • $\sigma(x) = 1/(1+e^{-x})$：sigmoid 函数，把任意实数压到 $(0, 1)$ 区间
    • $-\log$：负对数。概率越接近 1，$-\log$ 越接近 0；概率越接近 0，$-\log$ 越大
    • $\Delta = r_w - r_l$：两个答案的分数差。差越大，偏好概率越接近 1

    来源：Bradley & Terry 1952 (BT 模型)；Christiano et al. 2017 + InstructGPT §3.2（用于 RM 训练）。

    训完 RM，它就能给任意 (prompt, 回答) 打分——这就是 RL 需要的 reward 信号。

    第 3 步：用 RL 训 policy

    有了 RM 当 reward 信号，就可以做 RL 训练 policy。流程：

    RL 训练循环：

    1. policy 看 prompt，生成一个回答
    2. RM 给这个回答打分（reward）
    3. policy 根据分数调整——分数高的回答概率变大，分数低的变小
    4. 回到第 1 步，继续

    多轮之后，policy 学会生成高分回答。

    这是 RL（强化学习）的核心：没有标准答案，只有 reward 信号。模型通过试错，慢慢学会生成高 reward 的回答。区别于 SFT 的"模仿标准答案"。

    展开：PG（策略梯度定理）完整推导（7 步）

    目标：让 policy 生成高 reward 的回答。形式化为最大化期望 reward $J(\theta)$。

    第 1 步：写目标函数

    policy 在状态 $s$ 下选 action $a$ 的概率是 $\pi_\theta(a|s)$，选到 $a$ 后获得 reward $r(a)$。期望 reward 是对所有 action 加权求和：

    $$J(\theta) = \sum_a \pi_\theta(a|s) \cdot r(a)$$

    这是期望的定义：每个 action 的概率乘以它的 reward，加起来。

    第 2 步：对 $\theta$ 求梯度

    我们要找 $\nabla_\theta J$（让 $J$ 上升最快的方向）。$r(a)$ 不依赖 $\theta$，只有 $\pi_\theta$ 依赖 $\theta$：

    $$\nabla_\theta J = \sum_a \nabla_\theta \pi_\theta(a|s) \cdot r(a)$$

    第 3 步：问题——直接算 $\nabla_\theta \pi$ 很难

    直接算 $\pi_\theta$ 对 $\theta$ 的导数很复杂（涉及 softmax 的 Jacobian 等）。而且要求遍历所有 action，词表几万 token 算不动。需要找一个等价但好算的形式。

    第 4 步：log 技巧

    关键观察：一个函数的梯度可以写成它自己乘以 log 的梯度。数学上：

    $$\nabla_\theta \pi_\theta = \pi_\theta \cdot \nabla_\theta \log \pi_\theta$$

    为什么？因为 $\nabla \log f = \nabla f / f$（链式法则），所以 $\nabla f = f \cdot \nabla \log f$。把 $f$ 换成 $\pi_\theta$ 就是上面的式子。

    第 5 步：代回去

    把 log 技巧代入第 2 步的式子：

    $$\nabla_\theta J = \sum_a \pi_\theta(a|s) \cdot \nabla_\theta \log \pi_\theta(a|s) \cdot r(a)$$

    现在 $\pi_\theta(a|s)$ 和 $\nabla_\theta \log \pi_\theta(a|s)$ 凑在一起，可以写成期望的形式。

    第 6 步：写成期望

    期望 $\mathbb{E}_{a \sim \pi}[f(a)] = \sum_a \pi(a) \cdot f(a)$。所以上式就是：

    $$\nabla_\theta J = \mathbb{E}_{a \sim \pi_\theta}\left[ r(a) \cdot \nabla_\theta \log \pi_\theta(a|s) \right]$$

    这是 PG 定理的核心形式。为什么这步重要：期望可以用采样估计——从 $\pi_\theta$ 采几个样本，用平均代替期望。不用遍历所有 action 了。

    第 7 步：推广到多步——用 $G$ 替换 $r$

    到目前为止，公式里的 $r(a)$ 是"这次 action 的即时 reward"。但很多场景要看未来——比如下棋，这一步的好坏要看后续很多步的 reward 累加。所以把即时 reward $r$ 推广成 return $G$：

    $$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots$$

    其中 $\gamma \in [0,1]$ 是折扣因子（未来的 reward 要打折扣，因为越远的不确定性越大）。$G$ 就是"从现在开始，未来所有 reward 的折现和"。

    本例是单步场景——只有"prompt → 一个 action → 一个 reward"，没有未来。所以未来 reward 都是 0，$G$ 就退化成这次 action 的即时 reward：

    $$G = r$$

    本例 RM 给 "2" 这个回答打 1.0 分，所以 $r = 1.0$，$G = r = 1.0$。单步简化下梯度结构和多步完全一样，只是数值更简单。

    最终公式：

    $$\nabla_\theta J = \mathbb{E}_{a \sim \pi_\theta}\left[ G \cdot \nabla_\theta \log \pi_\theta(a|s) \right]$$

    实际训练时用单样本估计：$\nabla_\theta J \approx G \cdot \nabla_\theta \log \pi_\theta(a|s)$，采一个 $a$ 就能算梯度。

    直觉：$G$ 是这次 action 好不好。$G$ 大（好 action）就让 $\log \pi(a|s)$ 上升更多（这个 action 的概率变大）。$G$ 小或负（差 action）就让 $\log \pi(a|s)$ 下降。

    关键符号解释：

    • $\pi_\theta(a|s)$：policy（参数为 $\theta$ 的模型）在状态 $s$ 下选 action $a$ 的概率
    • $r(a)$：action $a$ 的 reward
    • $G$：return，未来 reward 的累计。单步场景 $G = r$
    • $\nabla_\theta$：对参数 $\theta$ 求梯度
    • $\log$：自然对数
    • $\mathbb{E}_{a \sim \pi_\theta}$：从 $\pi_\theta$ 采样 $a$ 的期望

    来源：Williams 1992 (REINFORCE)；Sutton & Barto 《Reinforcement Learning》§13（Policy Gradient Theorem）。

    on-policy vs off-policy

    PG 推导完了，但留下一个根本痛点：PG 是 on-policy 的。在讲 PPO 怎么改良之前，先把 on-policy 和 off-policy 这两个 RL 核心概念讲清楚——它们是 PG 的根本局限，也是 PPO 改良的出发点。

    on-policy vs off-policy（RL 的核心分类）

    区分标准：采样用的策略和正在更新的策略是不是同一个。

    on-policy（同策略）：采样和更新用同一个 policy。当前 $\pi_\theta$ 采样 → 算梯度更新 $\pi_\theta$ → $\theta$ 一变旧样本就失效（旧样本是从旧 $\pi_\theta$ 采的，不再代表新 policy 的行为分布）→ 必须用新 $\pi_\theta$ 重新采样。采一次用一次就丢。

    off-policy（异策略）：采样用一个 policy（behavior policy $\pi_{\text{old}}$），更新另一个 policy（target policy $\pi_\theta$）。用 importance ratio $r_t = \pi_\theta / \pi_{\text{old}}$ 修正分布偏移，旧数据可以反复用。

    PG 是 on-policy 的痛点：PG 要求期望 $\mathbb{E}_{a \sim \pi_\theta}$ 必须从当前 $\pi_\theta$ 采样。每次 $\theta$ 一更新，旧样本就失效——必须重新采样。这意味着采一批样本只能做一次梯度更新就丢。对 LLM 来说采一批样本要 forward 整个模型几十次，太贵了。这是 PG 在 LLM 场景下根本用不起来的核心原因，也是 PPO 改良的直接动机。

    PPO：PG 的改良版

    PPO 解决 PG 的两个痛点：

    1. 太浪费样本：PG 是 on-policy——采一次用一次就丢。每次更新后策略变了，旧样本就失效
    2. 可能被骗：如果某次 policy 乱答但 RM 给高分（reward hacking），policy 会一直乱答。PG 没有保护机制

    PPO 的两个关键改良：

    PPO 的两个关键：

    • 复用旧样本：用 importance ratio（$r_t = \pi_\text{new} / \pi_\text{old}$）让旧样本也能参与训练。省数据
    • clip（截断）：限制 $r_t$ 在 $[1-\epsilon, 1+\epsilon]$ 范围（通常 $\epsilon = 0.2$）。超出范围梯度消失——相当于自动刹车，防止 policy 偏太远

    直觉：允许 policy 往高分方向调整，但不能一步跳太远。如果发现某次"乱答"也能骗到高分，clip 说"先别太激进，最多允许偏离 20%"，超出就停止鼓励。

    为什么还需要 ref_policy 和 KL 约束：到这里 PPO 的两个改良讲完了，但伪代码里还会出现两个概念凭空出现：ref_policy 和 KL 散度。它们解决的是另一个问题——policy 训练时可能完全跑偏。

    具体场景：RM 是用一个有限偏好数据集训出来的，它对训练分布外的回答打分可能很不准。PPO 训练时 policy 会探索各种新回答，可能找到某种"乱码回答"骗到 RM 高分（reward hacking）。如果只追 RM 分数，policy 可能学到这种乱码策略，离原始 SFT 模型越来越远——这个模型就废了。

    解决方案：在 PPO 的 reward 里加一个 KL 惩罚项，限制 policy 不要离一个 ref_policy（参考策略）太远。ref_policy 通常是 SFT 后的模型副本，在训练开始时冻结——参数不再更新，当固定基准。

    KL 散度是什么

    KL 散度（Kullback-Leibler divergence）衡量两个概率分布的差异。对两个离散分布 $P$ 和 $Q$：

    $$D_{KL}(P \| Q) = \sum_i P(i) \log \frac{P(i)}{Q(i)}$$

    直觉：$D_{KL} = 0$ 表示两个分布完全一样；$D_{KL}$ 越大，差异越大。不对称——$D_{KL}(P \| Q) \neq D_{KL}(Q \| P)$。

    在 PPO 里，$P$ 是当前 policy $\pi_\theta$，$Q$ 是 ref_policy $\pi_{\text{ref}}$。$D_{KL}(\pi_\theta \| \pi_{\text{ref}})$ 越大，policy 离 ref 越远——这就是要惩罚的情况。

    具体怎么加惩罚：在 reward 里减去一个 KL 项：

    $$\text{reward}_{\text{final}} = \text{reward}_{\text{RM}} - \beta \cdot D_{KL}(\pi_\theta \| \pi_{\text{ref}})$$

    其中 $\beta$ 是 KL 系数（通常 0.001~0.1）。$\beta$ 越大，对偏离 ref 的惩罚越重，policy 越保守；$\beta$ 越小，policy 越自由探索。

    为什么 ref_policy 要冻结：如果 ref_policy 也跟着 policy 一起更新，那 KL 约束就失效了——policy 跑偏时 ref_policy 也跟着跑偏，KL 永远是 0，约束形同虚设。所以 ref_policy 必须在训练开始时冻结，整个 PPO 训练过程中参数不变。它代表"原始 SFT 模型的行为分布"，是 policy 不能偏离太远的锚点。

    实际实现：通常用每个 token 的 log prob 差近似 KL（避免显式算分布）：

    $$\text{kl}_t \approx \log \pi_\theta(a_t|s_t) - \log \pi_{\text{ref}}(a_t|s_t)$$

    这是单样本估计，期望下等价于真实的 KL。代码里 kl = new_log_probs - ref_log_probs 就是这个。

    关键直觉：ref_policy 是 policy 的"起跑线"。KL 惩罚相当于告诉 policy："你可以探索，但不能离起跑线太远"。这样既能用 RM 信号往高分方向调整，又防止跑偏到乱码策略。

    展开：PPO clip 目标完整推导（7 步）

    目标：从 PG 出发，解决两个问题——让旧样本能复用，且防止 policy 偏太远。

    第 1 步：回顾 PG 的目标

    PG 要最大化的是期望 return：

    $$J(\theta) = \mathbb{E}_{a \sim \pi_\theta}[G \cdot \log \pi_\theta(a|s)]$$

    问题：这个期望要求从当前 $\pi_\theta$ 采样。每次 $\theta$ 变了，旧样本就失效（on-policy）。能不能用旧策略 $\pi_{\theta_\text{old}}$ 采的样本？

    第 2 步：重要性采样——用旧样本估新策略的期望

    重要性采样的数学原理：想算 $\mathbb{E}_{x \sim p}[f(x)]$，但只有从 $q$ 采的样本。可以乘一个修正因子 $p(x)/q(x)$：

    $$\mathbb{E}_{x \sim p}[f(x)] = \mathbb{E}_{x \sim q}\left[\frac{p(x)}{q(x)} f(x)\right]$$

    证明：$\mathbb{E}_{x \sim q}\left[\frac{p(x)}{q(x)} f(x)\right] = \int q(x) \cdot \frac{p(x)}{q(x)} f(x) dx = \int p(x) f(x) dx = \mathbb{E}_{x \sim p}[f(x)]$。

    代入 PG：$p = \pi_\theta$（新策略），$q = \pi_{\theta_\text{old}}$（旧策略），$f = G \cdot \nabla_\theta \log \pi_\theta$：

    $$\nabla_\theta J = \mathbb{E}_{a \sim \pi_{\theta_\text{old}}}\left[\frac{\pi_\theta(a|s)}{\pi_{\theta_\text{old}}(a|s)} G \cdot \nabla_\theta \log \pi_\theta(a|s)\right]$$

    第 3 步：定义 importance ratio $r_t$

    把分数比简记为：

    $$r_t(\theta) = \frac{\pi_\theta(a|s)}{\pi_{\theta_\text{old}}(a|s)}$$

    直觉：$r_t = 1$ 表示新旧策略一样；$r_t > 1$ 表示新策略更偏好这个 action；$r_t < 1$ 表示更不偏好。

    代入第 2 步的式子：

    $$\nabla_\theta J = \mathbb{E}_{a \sim \pi_{\theta_\text{old}}}\left[ r_t \cdot G \cdot \nabla_\theta \log \pi_\theta(a|s) \right]$$

    现在用 log 技巧反过来——$\nabla_\theta \log \pi_\theta = \nabla_\theta \pi_\theta / \pi_\theta$（因为 $\nabla \log f = \nabla f / f$，把 $f$ 换成 $\pi_\theta$）。代入：

    $$\nabla_\theta J = \mathbb{E}_{a \sim \pi_{\theta_\text{old}}}\left[ r_t \cdot G \cdot \frac{\nabla_\theta \pi_\theta(a|s)}{\pi_\theta(a|s)} \right]$$

    把 $r_t / \pi_\theta$ 合并。$r_t = \pi_\theta / \pi_{\theta_\text{old}}$，所以：

    $$\frac{r_t}{\pi_\theta} = \frac{\pi_\theta / \pi_{\theta_\text{old}}}{\pi_\theta} = \frac{1}{\pi_{\theta_\text{old}}}$$

    代入：

    $$\nabla_\theta J = \mathbb{E}_{a \sim \pi_{\theta_\text{old}}}\left[ \frac{G}{\pi_{\theta_\text{old}}(a|s)} \cdot \nabla_\theta \pi_\theta(a|s) \right]$$

    展开期望（$\mathbb{E}_{a \sim \pi_{\theta_\text{old}}}[f(a)] = \sum_a \pi_{\theta_\text{old}}(a|s) \cdot f(a)$）：

    $$\nabla_\theta J = \sum_a \pi_{\theta_\text{old}}(a|s) \cdot \frac{G}{\pi_{\theta_\text{old}}(a|s)} \cdot \nabla_\theta \pi_\theta(a|s)$$

    $\pi_{\theta_\text{old}}$ 在分子分母约掉了：

    $$\nabla_\theta J = \sum_a G \cdot \nabla_\theta \pi_\theta(a|s) = \nabla_\theta \sum_a G \cdot \pi_\theta(a|s) = \nabla_\theta \mathbb{E}_{a \sim \pi_\theta}[G]$$

    所以 $J(\theta) = \mathbb{E}_{a \sim \pi_\theta}[G]$。用 $r_t$ 表示成"从旧策略采样"的形式（这样旧样本才能复用）：

    $$J(\theta) = \mathbb{E}_{a \sim \pi_{\theta_\text{old}}}\left[ r_t(\theta) \cdot G \right]$$

    验证等价：展开右边 $\sum_a \pi_{\theta_\text{old}} \cdot r_t \cdot G = \sum_a \pi_{\theta_\text{old}} \cdot \frac{\pi_\theta}{\pi_{\theta_\text{old}}} \cdot G = \sum_a \pi_\theta \cdot G = \mathbb{E}_{a \sim \pi_\theta}[G]$。对上了。

    这就是用旧样本估计新策略的期望 return。看起来可以直接最大化了，但有问题。

    第 4 步：问题——$r_t$ 太大方差爆炸

    如果 $\pi_\theta$ 离 $\pi_{\theta_\text{old}}$ 很远（$r_t$ 很大或很小），估计的方差会爆炸，训练不稳定。需要限制 $\theta$ 不要离 $\theta_\text{old}$ 太远。

    TRPO（PPO 的前身）用 KL 散度硬约束，但计算复杂。PPO 用一个更简单的办法——clip。

    第 5 步：用 advantage $A$ 替换 $G$

    先做一个小改进：把 $G$ 换成 advantage $A = G - V(s)$（减去 baseline $V(s)$）。$V(s)$ 是状态 $s$ 的"预期未来 return"。

    为什么要减？$G$ 的绝对值波动大，方差大。减去 baseline 不改变梯度的期望（因为 $\mathbb{E}[\nabla \log \pi \cdot V(s)] = 0$），但能降低方差。$A > 0$ 表示这个 action 比平均好，$A < 0$ 表示比平均差。

    本例单步简化：$V(s) = 0$，$A = G = r$。

    目标变成：

    $$J(\theta) = \mathbb{E}\left[ r_t(\theta) \cdot A \right]$$

    这里引出了一堆概念：advantage、baseline、critic、value、returns、GAE。它们其实是一条链——为了让 advantage 估计更准、训练更稳而引入的。先继续往下推 clip，下一个 details 单独展开讲它们。

    第 6 步：clip 限制 $r_t$

    PPO 的核心创新：把 $r_t$ 截断到 $[1-\epsilon, 1+\epsilon]$ 范围内（通常 $\epsilon = 0.2$）。超出范围的梯度消失——相当于自动刹车。

    定义 clip 函数：

    $$\text{clip}(r_t, 1-\epsilon, 1+\epsilon) = \begin{cases} 1-\epsilon & \text{if } r_t < 1-\epsilon \\ r_t & \text{if } 1-\epsilon \leq r_t \leq 1+\epsilon \\ 1+\epsilon & \text{if } r_t > 1+\epsilon \end{cases}$$

    第 7 步：取 min——做悲观估计

    PPO 目标取 $r_t A$ 和 $\text{clip}(r_t) A$ 的较小者：

    $$L^{\text{CLIP}}(\theta) = \mathbb{E}\left[ \min\left( r_t(\theta) A, \, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) A \right) \right]$$

    为什么要取 min？分情况看：

    • 当 $A > 0$（好 action）：要鼓励 $r_t$ 增大。但 clip 把 $r_t$ 封顶在 $1+\epsilon$。取 min 后，当 $r_t > 1+\epsilon$ 时 $L = (1+\epsilon)A$ 不再增加——梯度消失，停止鼓励。防止 policy 过度偏好这个 action。
    • 当 $A < 0$（差 action）：要鼓励 $r_t$ 减小。但 clip 把 $r_t$ 托底在 $1-\epsilon$。取 min 后，当 $r_t < 1-\epsilon$ 时 $L = (1-\epsilon)A$（注意 $A$ 是负的，$(1-\epsilon)A > r_t A$，取 min 取到 $r_t A$，这个方向梯度也消失）——停止惩罚。

    两边都封住，$r_t$ 在 $[1-\epsilon, 1+\epsilon]$ 范围内梯度正常，超出范围梯度消失。这就是 clip 的保护机制——允许 policy 变好，但不能一步跳太远。

    最终公式：

    $$L^{\text{CLIP}}(\theta) = \mathbb{E}\left[ \min\left( r_t(\theta) A, \, \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) A \right) \right]$$

    训练时最大化 $L^{\text{CLIP}}$（等价于最小化 $-L^{\text{CLIP}}$）。

    关键符号解释：

    • $\pi_\theta$：新策略（正在训练的 policy）
    • $\pi_{\theta_\text{old}}$：旧策略（采样时用的，固定不更新）
    • $r_t(\theta) = \pi_\theta(a|s) / \pi_{\theta_\text{old}}(a|s)$：importance ratio，新旧策略对同一个 action 的偏好倍数
    • $A$：advantage，$A = G - V(s)$，action 相对平均好不好
    • $\epsilon$：clip 范围，通常 0.1~0.3，本例 0.2
    • $\text{clip}(x, a, b)$：把 $x$ 截到 $[a, b]$
    • $\min$：取较小者，做悲观估计

    来源：Schulman et al. 2017 (PPO 论文)；TRPO（前身）← Schulman et al. 2015。

    展开：advantage、baseline、critic、value、GAE、returns 都是什么（8 个点）

    第 5 步凭空出现了一堆概念：advantage、baseline、critic、value、GAE、returns。它们其实是一条链，下面从头讲清楚。

    1. baseline 是什么

    直接用 reward 当梯度权重有个问题：方差大。假设某个 prompt 平均能拿 100 分，某次 action 拿了 95 分——95 看起来是高分，但实际比平均低，应该惩罚而不是奖励。直接用 reward 分不清这种情况。

    解决方案：减去一个 baseline（平均分）。baseline 是"这个 prompt 平均能拿多少分"。减完后：

    $$A = G - V(s)$$

    这里 $A$ 就是 advantage（优势），$G$ 是 return（这次 action 实际拿的分），$V(s)$ 是 baseline（预期平均分）。

    • $A > 0$：这次 action 比平均好 → 鼓励 policy 多做
    • $A < 0$：比平均差 → 惩罚，让 policy 少做
    • $A = 0$：和平均一样 → 无所谓

    2. value（$V$）是什么

    $V(s)$ 就是"状态 $s$ 的预期 return"——预测"如果用当前 policy，从 $s$ 开始平均能拿多少 reward"。

    问题：怎么知道每个 prompt 的平均分？没法事先知道，要边训边估。于是训一个 value model（也叫 critic）专门预测 $V(s)$——它和 policy（actor）是两个独立的模型，actor 学会生成好回答，critic 学会预测"这个 prompt 平均能拿多少分"。

    3. critic 怎么训

    critic 的目标是学会预测 return。它的训练 target 是实际看到的 return——也就是 returns。

    $$\text{value\_loss} = (V_\phi(s) - \text{returns})^2$$

    critic 的预测值 $V_\phi(s)$ 越接近实际 returns，value_loss 越小。critic 越准，advantage 估计越准，policy 训练越稳。

    4. returns 是什么

    returns 是"实际看到的累计 reward"——critic 要拟合的目标。

    • 单步场景（本例）：returns = reward（只有一个 reward，没有未来）
    • 多步场景：returns = $r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots$（未来所有 reward 的折现累计）

    这里 $\gamma \in [0, 1]$ 是折扣因子——未来的 reward 要打折扣，因为越远越不确定。$\gamma = 1$ 表示不折扣（未来和现在一样重要），$\gamma = 0.99$ 是常用值。

    5. 单步场景下 advantage 和 returns 的关系

    单步场景（1 prompt → 1 response → 1 reward）下，没有"未来"，所以：

    $$A = G - V = \text{reward} - V$$

    $$\text{returns} = A + V = \text{reward}$$

    单步下 returns 就是 reward 本身，但保留这个抽象是为了和多步场景统一接口。

    6. GAE 是什么（多步场景的 advantage 算法）

    多步场景下，reward 只在最后给（或分散给一些 token），但每个 token 都"贡献"了结果。哪个 token 该奖哪个该罚？

    GAE（Generalized Advantage Estimation，广义优势估计）解决这个问题。它用 TD error（时序差分）从后往前递推算每个时刻的 advantage：

    $$\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t) \quad \text{(TD error，实际 - 预期)}$$

    $$A_t^{\text{GAE}} = \delta_t + (\gamma\lambda)\delta_{t+1} + (\gamma\lambda)^2\delta_{t+2} + \cdots$$

    这里 $\lambda \in [0, 1]$ 控制"要看多远的未来"：

    • $\lambda = 0$：只看当前一步 TD error，偏差大、方差小
    • $\lambda = 1$：用所有未来 reward，偏差小、方差大
    • $\lambda = 0.95$（常用）：折中

    本代码单步场景下 GAE 退化：只有一步，没有未来，所以 $\delta = \text{reward} - V$，GAE 递推只有一项，advantage = reward - V。代码里的 compute_gae 函数就简化成了 advantages = rewards - values。

    7. advantage 归一化（工程实践）

    训练时通常对 advantage 做标准化：

    $$A \leftarrow \frac{A - \text{mean}(A)}{\text{std}(A) + \epsilon}$$

    把 advantage 缩放到均值 0、方差 1。降方差，让训练更稳。这是工程 trick，不影响数学正确性。

    8. .detach() 是什么

    PyTorch 的概念。算 old_log_probs 时，它依赖当时的 actor 参数 $\theta$，会建立计算图。但训练时 $\theta$ 已经变了，old_log_probs 应该当常数用——不能让它参与梯度反传（否则梯度方向会乱）。

    .detach() 把 tensor 从计算图里"摘出来"，断开和 $\theta$ 的连接。这样 backward 时 old_log_probs 当作固定值，不传梯度。

    等价的写法：with torch.no_grad(): 包住整个块——从一开始就不建计算图。代码里 ref_policy 和 critic 用 no_grad（本来就冻结），old_log_probs 用 detach（需要它的值参与 loss，但不要梯度）。

    关键符号解释：

    • $V(s)$ / value：状态 $s$ 的预期 return（critic 预测）
    • $A$ / advantage：$A = G - V(s)$，这次 action 相对平均好不好
    • returns：实际看到的累计 reward，critic 的训练 target
    • critic / value model：预测 $V(s)$ 的独立模型，和 actor 一起训
    • $\gamma$：折扣因子，未来 reward 的折扣权重
    • $\lambda$：GAE 参数，控制看多远的未来
    • GAE：多步场景算 advantage 的算法，单步下退化成 $A = r - V$
    • value_loss：critic 的 loss，让预测 $V$ 接近实际 returns
    • 归一化 advantage：工程 trick，降方差
    • detach：PyTorch 概念，把 tensor 从计算图断开，当常数用

    来源：GAE ← Schulman et al. 2015《High-Dimensional Continuous Control Using Generalized Advantage Estimation》；critic/value 是 RL 领域常识，见 Sutton & Barto 《Reinforcement Learning》。

    PPO 是 RLHF 的事实标准。ChatGPT、Claude、Gemini 这些模型的 RLHF 阶段都用 PPO。

    训练流程伪代码

    上面讲了三步流程的数学原理。下面用伪代码展示工业界实际怎么训——RM 训练和 PPO 训练各一个循环。代码省略工程细节（batch、混合精度、分布式等），只展示核心逻辑。

    展开：训 RM 的训练循环伪代码 
    # 数据：偏好对 (prompt, chosen, rejected)
# RM 是一个 transformer，输入 (prompt, response) 输出标量分数

for prompt, chosen, rejected in preference_dataset:
    # 前向：RM 给两个回答打分
    r_w = RM(prompt, chosen)       # 好答案的分数
    r_l = RM(prompt, rejected)     # 差答案的分数

    # RM loss = -log sigmoid(r_w - r_l)（Bradley-Terry）
    loss = -log_sigmoid(r_w - r_l)

    # 反传 + 更新 RM 参数
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()

# 训完的 RM 冻结，后面当 reward 信号用
RM.eval()
for p in RM.parameters():
    p.requires_grad = False

    这一步和 SFT 训练循环长得几乎一样——只是 loss 从交叉熵换成 BT loss，数据从 (prompt, 标准答案) 换成 (prompt, 好答案, 差答案)。

    下面的 PPO 训练伪代码参考 OpenRLHF 的实现，省略工程细节（分布式、混合精度、梯度累积等），只展示核心逻辑。关键是看出 PPO 和 PG 的区别：采样和训练分离，一批样本训练多个 epoch——这就是 off-policy 改良的体现。

    展开：用 PPO 训 policy 的训练循环伪代码（参考 OpenRLHF） 
    # ============ 模型准备 ============
# actor        = SFT 后的 LLM（待训练，参数 θ）
# ref_policy   = actor 的冻结副本（KL 约束基准，不更新）
# critic       = 价值模型，预测每个 token 的预期 return（当 baseline）
# reward_model = 第 2 步训好的 RM（冻结）

# ============ 超参 ============
# rollout_batch_size = 一次采样的 prompt 数（如 512）
# num_epochs         = 一批样本训练几个 epoch（如 4）—— PPO 能复用旧样本的关键
# mini_batch_size    = 训练时的 mini-batch 大小（如 64）
# clip_eps           = 0.2, gamma = 1.0, lambd = 1.0, kl_coef = 0.001

# ========================================
# 外层循环：每个 episode = 一次采样 + 多 epoch 训练
# ========================================
for episode in range(num_episodes):
    prompts = sample_prompts(prompt_dataset, batch_size=rollout_batch_size)

    # ---------- 阶段 1：Rollout（采样 + 算经验）----------
    # 用当前 actor 采一批样本。采完这批 θ 就冻结，
    # 后面多个 epoch 都用这批样本训练——这是 PPO 的核心改良
    experiences = []
    for prompt in prompts:
        # 1.1 当前 actor 生成 response
        response = actor.generate(prompt)

        # 1.2 算旧策略的 log prob（采样时的 actor，后面训练时不重新采样）
        old_log_probs = actor.log_prob(prompt, response)
        # 1.3 ref_policy 的 log prob（用于 KL 约束）
        ref_log_probs = ref_policy.log_prob(prompt, response)
        # 1.4 critic 预测每个 token 的 value
        old_values = critic.value(prompt, response)
        # 1.5 RM 打分（只在最后一个 token 给 reward）
        reward = reward_model(prompt, response)
        # 1.6 KL 惩罚：每个 token 的 reward = -kl_coef * KL + 最后 token 的 RM reward
        kl = old_log_probs - ref_log_probs
        token_rewards = -kl_coef * kl
        token_rewards[-1] += reward

        # 1.7 GAE 算 advantage 和 return（从后往前递推）
        advantages, returns = compute_gae(old_values, token_rewards, gamma, lambd)

        experiences.append({
            prompt, response, old_log_probs, old_values,
            advantages, returns
        })

    # advantage 归一化（工程实践，降方差）
    advantages = normalize(advantages)

    # ---------- 阶段 2：PPO 训练（一批样本训 num_epochs 轮）----------
    # 这里体现 off-policy：用 old_log_probs（采样时的 actor）
    # 训练时 θ 已经变了，用 importance ratio 修正
    for epoch in range(num_epochs):
        for batch in split_into_minibatches(experiences, mini_batch_size):
            # ===== Actor 更新（policy loss）=====
            # 当前 actor 重新算 log prob（θ 已经变了）
            new_log_probs = actor.log_prob(batch.prompt, batch.response)
            # importance ratio = π_new / π_old
            ratio = exp(new_log_probs - batch.old_log_probs)

            # clipped surrogate loss（PPO 核心）
            surr1 = ratio * batch.advantages
            surr2 = clamp(ratio, 1-clip_eps, 1+clip_eps) * batch.advantages
            actor_loss = -mean(min(surr1, surr2))

            # KL 惩罚（防止 actor 离 ref_policy 太远）
            kl = new_log_probs - ref_log_probs
            actor_loss += kl_coef * mean(kl)

            actor_loss.backward()
            actor_optimizer.step()
            actor_optimizer.zero_grad()

            # ===== Critic 更新（value loss）=====
            # 训 critic 学会预测 return（让 baseline 更准）
            new_values = critic.value(batch.prompt, batch.response)
            # clipped value loss（和 policy loss 类似，clip 防止更新太激进）
            values_clipped = batch.old_values + clamp(
                new_values - batch.old_values, -clip_eps, clip_eps)
            value_loss = 0.5 * mean(max(
                (new_values - batch.returns) ** 2,
                (values_clipped - batch.returns) ** 2))

            value_loss.backward()
            critic_optimizer.step()
            critic_optimizer.zero_grad()

    # 一批样本训完，进入下一个 episode 重新采样
    # 新 episode 的 old_log_probs 会用新的 actor 重新算

    伪代码里出现的 PyTorch / 工程概念速查：

    • .detach()：把 tensor 从计算图摘出来，当常数用。old_log_probs 是采样时旧策略算的，训练时要当固定值（不能让它的梯度影响当前 θ），所以 detach。等价写法：with torch.no_grad(): 包整个块（ref_policy 和 critic 用这种，因为它们本来就冻结）。
    • torch.cat：拼接。把 rollout 时逐条采的 8 条经验拼成一个 batch，方便并行处理。
    • advantage 归一化：把 advantage 缩放到均值 0、方差 1（(A - mean) / std）。降方差，让训练更稳。工程 trick，不影响数学正确性。
    • value_loss：critic 的 loss，让预测的 $V(s)$ 接近实际 returns。通常用 MSE，PPO 还加 clip 防止 critic 更新太激进。
    • torch.multinomial：按概率分布采样。给 [0.1, 0.7, 0.2]，按概率采一个索引（70% 概率采到 1）。训练时用这个采样新回答（带探索）。
    • log_softmax vs softmax：采样时用 softmax 得到概率喂给 multinomial；训练时用 log_softmax 直接拿 log 概率（数值更稳，避免 log(0)）。

    关键观察：on-policy → off-policy 在代码哪里体现：

    • 采样的位置：阶段 1 用当前 actor 采一批 response，算 old_log_probs。这一步是 on-policy 的——采的样本代表当前策略的行为分布
    • 训练的位置：阶段 2 训练时 actor 的 θ 已经变了，但不重新采样。用 ratio = exp(new_log_probs - old_log_probs) 修正分布偏移——这就是 off-policy 的体现
    • 样本复用的关键：一批样本训练 num_epochs 轮（通常 4）。PG 只能训 1 轮就丢，PPO 能训 4 轮——样本效率提升 4 倍
    • clip 的作用：训练几个 epoch 后 ratio 会偏离 1 越来越远。clip 把它限制在 [1-ε, 1+ε] 范围，超出就停止鼓励——防止 θ 偏离太远导致修正失效

    对比 PG：PG 是 采一批 → 训 1 轮 → 丢掉 → 重新采，每次 θ 变了旧样本就失效。PPO 是 采一批 → 训 4 轮 → 丢掉 → 重新采，4 轮都用 importance ratio 修正，样本利用率高 4 倍。

    这是 RLHF 训练的核心循环。和 SFT/DPO 相比，工程复杂度高很多——要同时维护 4 个模型在显存里（policy、ref_policy、value_model、RM），还要算 importance ratio 和 clip。这也是为什么 DPO 流行：去掉 ref_policy、value_model、RM，只留 policy 一个模型，循环和 SFT 几乎一样。

    显存瓶颈 + GRPO 改进：

    4 个 LLM 同时在显存里，参数+梯度+优化器状态，显存需求约是 SFT 的 4 倍。这是 RLHF 工程上的最大挑战，也是为什么开源界更多用 DPO。

    GRPO 的改进：去掉 value_model（少一个模型），改用组内 reward 的相对值当 baseline。DeepSeek-R1 用的就是 GRPO。详见 GRPO 概念文档。

    场景：词表 5 个 token，每个 prompt 是 1 个 token，response 也是 1 个 token。reward 由一个假的 reward_model 给出（手工设定的分数表）。目标是让 policy 学会生成高 reward 的 token。

    展开：完整 ppo_example.py 代码（可直接复制运行） 
    """
PPO 最小可运行实现 — 对应 content/concepts/rlhf/index.html 里的伪代码

这是一个教学版实现，省略了工程细节（分布式、混合精度、大模型），
但保留了 PPO 的核心结构：
  1. Rollout（采样 + 算经验）
  2. Train（多 epoch 训练 + importance ratio + clip）

运行：python3 content/concepts/rlhf/ppo_example.py
依赖：torch（已安装）

场景：词表 5 个 token，每个 prompt 是 1 个 token，response 也是 1 个 token。
reward 由一个假的 reward_model 给出（手工设定的分数表）。
目标是让 policy 学会生成高 reward 的 token。
"""

import torch
import torch.nn as nn
import torch.nn.functional as F
from copy import deepcopy
from dataclasses import dataclass
from typing import List
import math

# ============================================================
# 配置
# ============================================================
VOCAB_SIZE = 5          # 词表大小（token 0..4）
EMBED_DIM = 16           # embedding 维度
HIDDEN_DIM = 32          # 隐藏层维度
PROMPT_TOKEN = 0         # 所有 prompt 都是 token 0
TARGET_TOKEN = 2         # 我们希望 policy 学会生成这个 token（reward 最高）

# PPO 超参
ROLLOUT_BATCH_SIZE = 8   # 每次（rollout）采多少个 prompt
NUM_EPOCHS = 4           # 每批数据训几个 epoch（PPO 核心：样本复用）
MINI_BATCH_SIZE = 4      # 训练时的 mini-batch 大小
CLIP_EPS = 0.2           # PPO clip 范围
KL_COEF = 0.001          # KL 惩罚系数
GAMMA = 1.0              # 折扣因子（单步场景，无未来）
LAMBDA = 1.0             # GAE 参数
LEARNING_RATE = 1e-2
NUM_EPISODES = 20        # 总共跑几个 episode

# 假的 reward 表：生成 token 2 给 1.0 分，其他给低分
REWARD_TABLE = torch.tensor([-0.5, 0.0, 1.0, 0.1, -0.2])


# ============================================================
# 模型定义
# ============================================================
class TinyLLM(nn.Module):
    """
    极简 LLM：embedding + 一个隐藏层 + 输出 logits
    输入一个 token，输出下一个 token 的 logits（词表大小）
    """
    def __init__(self, vocab_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.fc1 = nn.Linear(embed_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, vocab_size)

    def forward(self, input_ids):
        """input_ids: (batch,) → logits: (batch, vocab_size)"""
        emb = self.embedding(input_ids)       # (B, embed_dim)
        h = F.relu(self.fc1(emb))             # (B, hidden_dim)
        logits = self.fc2(h)                  # (B, vocab_size)
        return logits

    @torch.no_grad()
    def generate(self, input_ids):
        """采样一个 token（贪婪解码，简化用）"""
        logits = self.forward(input_ids)      # (B, vocab_size)
        probs = F.softmax(logits, dim=-1)
        # 用多项分布采样（训练时要探索）
        token = torch.multinomial(probs, num_samples=1)
        return token.squeeze(-1)              # (B,)

    def log_prob(self, input_ids, response):
        """算 log π(response | input_ids)，response 是采样的 token"""
        logits = self.forward(input_ids)       # (B, vocab_size)
        log_probs = F.log_softmax(logits, dim=-1)
        # 取出 response 对应的 log prob
        return log_probs.gather(-1, response.unsqueeze(-1)).squeeze(-1)


class ValueModel(nn.Module):
    """极简 value model：和 actor 结构一样，但输出标量"""
    def __init__(self, vocab_size, embed_dim, hidden_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim)
        self.fc1 = nn.Linear(embed_dim, hidden_dim)
        self.fc2 = nn.Linear(hidden_dim, 1)

    def forward(self, input_ids):
        emb = self.embedding(input_ids)
        h = F.relu(self.fc1(emb))
        value = self.fc2(h).squeeze(-1)       # (B,)
        return value


# ============================================================
# 经验数据结构
# ============================================================
@dataclass
class Experience:
    prompt: torch.Tensor      # (B,) prompt token
    response: torch.Tensor    # (B,) response token
    old_log_probs: torch.Tensor   # (B,) 采样时 actor 的 log π
    ref_log_probs: torch.Tensor   # (B,) ref_policy 的 log π
    old_values: torch.Tensor      # (B,) critic 的 value
    reward: torch.Tensor          # (B,) RM 给的 reward
    advantage: torch.Tensor = None
    returns: torch.Tensor = None


# ============================================================
# GAE（Generalized Advantage Estimation）
# ============================================================
def compute_gae(values, rewards, gamma=GAMMA, lambd=LAMBDA):
    """
    单步场景简化版 GAE：advantage = reward - value
    （多步场景要从后往前递推，这里只有一步所以直接算）
    """
    advantages = rewards - values
    returns = advantages + values  # R = A + V
    return advantages, returns


# ============================================================
# Reward Model（假的，用固定分数表）
# ============================================================
def reward_model(prompt, response):
    """假的 RM：根据 response token 查表给分"""
    return REWARD_TABLE[response]


# ============================================================
# PPO 训练器
# ============================================================
class PPOTrainer:
    def __init__(self):
        # actor = 待训练的 policy
        self.actor = TinyLLM(VOCAB_SIZE, EMBED_DIM, HIDDEN_DIM)
        # ref_policy = 冻结的副本（KL 基准）
        self.ref_policy = deepcopy(self.actor)
        for p in self.ref_policy.parameters():
            p.requires_grad = False
        # critic = value model（待训练）
        self.critic = ValueModel(VOCAB_SIZE, EMBED_DIM, HIDDEN_DIM)

        self.actor_optim = torch.optim.Adam(self.actor.parameters(), lr=LEARNING_RATE)
        self.critic_optim = torch.optim.Adam(self.critic.parameters(), lr=LEARNING_RATE)

    def make_experience(self, prompt_token):
        """阶段 1：采样一个 (prompt, response) 并算经验"""
        prompt = torch.tensor([prompt_token])
        # 1.1 当前 actor 生成 response
        response = self.actor.generate(prompt)
        # 1.2 旧策略的 log prob（detach，不参与梯度）
        old_log_probs = self.actor.log_prob(prompt, response).detach()
        # 1.3 ref_policy 的 log prob（用于 KL）
        with torch.no_grad():
            ref_log_probs = self.ref_policy.log_prob(prompt, response)
        # 1.4 critic 预测 value
        with torch.no_grad():
            old_values = self.critic(prompt)
        # 1.5 RM 打分
        reward = reward_model(prompt, response)
        # 1.6 GAE 算 advantage（单步：A = r - V）
        advantages, returns = compute_gae(old_values, reward)
        return Experience(
            prompt=prompt, response=response,
            old_log_probs=old_log_probs,
            ref_log_probs=ref_log_probs,
            old_values=old_values,
            reward=reward,
            advantage=advantages,
            returns=returns,
        )

    def ppo_train(self, experiences: List[Experience]):
        """阶段 2：PPO 训练（多 epoch + importance ratio + clip）"""
        # 拼成 batch
        prompts = torch.cat([e.prompt for e in experiences])
        responses = torch.cat([e.response for e in experiences])
        old_log_probs = torch.cat([e.old_log_probs for e in experiences])
        ref_log_probs = torch.cat([e.ref_log_probs for e in experiences])
        old_values = torch.cat([e.old_values for e in experiences])
        advantages = torch.cat([e.advantage for e in experiences])
        returns = torch.cat([e.returns for e in experiences])

        # advantage 归一化
        advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)

        stats = {"policy_loss": 0, "value_loss": 0, "kl": 0, "n_steps": 0}

        for epoch in range(NUM_EPOCHS):
            # 打乱数据
            n = len(experiences)
            perm = torch.randperm(n)
            for i in range(0, n, MINI_BATCH_SIZE):
                idx = perm[i:i+MINI_BATCH_SIZE]
                mb_prompts = prompts[idx]
                mb_responses = responses[idx]
                mb_old_log_probs = old_log_probs[idx]
                mb_ref_log_probs = ref_log_probs[idx]
                mb_old_values = old_values[idx]
                mb_advantages = advantages[idx]
                mb_returns = returns[idx]

                # ===== Actor 更新 =====
                # 当前 actor 重新算 log prob（θ 已经变了）
                new_log_probs = self.actor.log_prob(mb_prompts, mb_responses)
                # importance ratio = π_new / π_old = exp(log π_new - log π_old)
                ratio = torch.exp(new_log_probs - mb_old_log_probs)

                # PPO clip loss
                surr1 = ratio * mb_advantages
                surr2 = torch.clamp(ratio, 1 - CLIP_EPS, 1 + CLIP_EPS) * mb_advantages
                policy_loss = -torch.min(surr1, surr2).mean()

                # KL 惩罚（防止 actor 离 ref_policy 太远）
                kl = (new_log_probs - mb_ref_log_probs).mean()
                policy_loss += KL_COEF * kl

                self.actor_optim.zero_grad()
                policy_loss.backward()
                self.actor_optim.step()

                # ===== Critic 更新 =====
                new_values = self.critic(mb_prompts)
                # clipped value loss
                values_clipped = mb_old_values + torch.clamp(
                    new_values - mb_old_values, -CLIP_EPS, CLIP_EPS)
                value_loss = 0.5 * torch.max(
                    (new_values - mb_returns) ** 2,
                    (values_clipped - mb_returns) ** 2
                ).mean()

                self.critic_optim.zero_grad()
                value_loss.backward()
                self.critic_optim.step()

                stats["policy_loss"] += policy_loss.item()
                stats["value_loss"] += value_loss.item()
                stats["kl"] += kl.item()
                stats["n_steps"] += 1

        return {k: v / max(stats["n_steps"], 1) for k, v in stats.items() if k != "n_steps"}

    @torch.no_grad()
    def evaluate(self):
        """评估：让 actor 贪婪生成 100 次，看生成 target_token 的概率"""
        prompt = torch.tensor([PROMPT_TOKEN])
        logits = self.actor(prompt)
        probs = F.softmax(logits, dim=-1)
        count = 0
        for _ in range(100):
            token = torch.multinomial(probs, num_samples=1).item()
            if token == TARGET_TOKEN:
                count += 1
        return count / 100, probs.squeeze().tolist()


# ============================================================
# 主训练循环
# ============================================================
def main():
    print("=" * 60)
    print("PPO 最小实现 — 让 policy 学会生成 token", TARGET_TOKEN)
    print("=" * 60)
    print(f"配置：vocab={VOCAB_SIZE}, rollout_batch={ROLLOUT_BATCH_SIZE}, "
          f"epochs={NUM_EPOCHS}, clip={CLIP_EPS}")
    print(f"reward 表：{REWARD_TABLE.tolist()}")
    print(f"目标：让 policy 生成 token {TARGET_TOKEN}（reward={REWARD_TABLE[TARGET_TOKEN].item()}）")
    print()

    trainer = PPOTrainer()

    # 训练前评估
    p_target, probs = trainer.evaluate()
    print(f"[训练前] 生成 token {TARGET_TOKEN} 的概率: {p_target:.2%}")
    print(f"         各 token 概率: {[f'{p:.3f}' for p in probs]}")
    print()

    # ============ 外层循环：每个 episode = 采样 + 多 epoch 训练 ============
    for episode in range(NUM_EPISODES):
        # ---------- 阶段 1：Rollout（采样）----------
        experiences = []
        for _ in range(ROLLOUT_BATCH_SIZE):
            exp = trainer.make_experience(PROMPT_TOKEN)
            experiences.append(exp)

        # 看一下这批 response 里有多少个 target token
        responses = [e.response.item() for e in experiences]
        hit = sum(1 for r in responses if r == TARGET_TOKEN)
        avg_reward = sum(e.reward.item() for e in experiences) / len(experiences)

        # ---------- 阶段 2：PPO 训练（多 epoch）----------
        stats = trainer.ppo_train(experiences)

        # 每 5 个 episode 打印一次
        if (episode + 1) % 5 == 0 or episode == 0:
            print(f"[Episode {episode+1:3d}] "
                  f"hit={hit}/{ROLLOUT_BATCH_SIZE} "
                  f"avg_reward={avg_reward:+.3f} "
                  f"policy_loss={stats['policy_loss']:+.4f} "
                  f"value_loss={stats['value_loss']:.4f} "
                  f"kl={stats['kl']:.4f}")

    print()
    # 训练后评估
    p_target, probs = trainer.evaluate()
    print(f"[训练后] 生成 token {TARGET_TOKEN} 的概率: {p_target:.2%}")
    print(f"         各 token 概率: {[f'{p:.3f}' for p in probs]}")
    print()
    if p_target > 0.5:
        print("✓ 训练成功！policy 学会了生成目标 token")
    else:
        print("✗ 训练未收敛，可以多跑几个 episode 或调大 learning rate")


if __name__ == "__main__":
    main()

    三、总结：RLHF 的两条路线

    本章你将知道：RLHF 不是唯一路线，DPO 是简化替代方案，什么时候用哪个。

    RLHF 完整流程 3 步（偏好数据 → 训 RM → 用 PPO 训 policy），复杂但能持续探索。有人发现：数学上可以跳过 RM 和 RL，直接在偏好数据上训 policy。这就是 DPO。

    维度RLHFDPO
    步数3 步1 步
    需要训 RM是否
    需要 RL是（PPO）否
    能持续探索是否（只用静态数据）
    训练稳定性较差好
    代表模型ChatGPT、ClaudeLlama 3、Qwen

    DPO 的核心洞察：最优的 policy 可以直接从偏好数据 + 参考模型算出来，不需要显式训 RM，也不需要 PPO。一个 loss 搞定。

    工业界现在流行：DPO 做 base，RLHF 做精细调整。两者不是对立的，可以组合用。
