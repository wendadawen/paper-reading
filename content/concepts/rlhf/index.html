    RLHF：从人类反馈学习

    问题：SFT（Supervised Fine-Tuning，监督微调）只能模仿"标准答案"，但写诗、对话、总结这类开放任务没有唯一正确答案，人类偏好没法用示范数据表达
    方案：RLHF（Reinforcement Learning from Human Feedback，从人类反馈学习）三步流程——SFT 起步 → 用人类偏好对训一个 RM（Reward Model，奖励模型，给回答打分）→ 把 RM 当环境用 PPO（一种强化学习算法）训 policy（模型本身），加 KL 约束（限制模型别偏离参考模型太远）防跑偏
    产物：理解 RLHF 三步流程、RM 怎么从偏好对训出来、PPO 目标里 KL 约束为什么必要、reward hacking（模型骗 RM 拿高分但回答差）怎么发生

    30 秒上下文：先知道它站在哪里

    大流程
    LLM 训练 = 预训练 → SFT → 对齐（RLHF 或 DPO）。RLHF 属于"对齐"层
    旧方案
    只做 SFT，给 prompt 配"标准答案"做最大似然，让模型模仿
    痛点
    开放任务无标准答案；人类擅长比较却难直接写出最好答案；SFT 学到的是模仿不是偏好
    改变点
    让人类当裁判（比好坏）而非当老师（写答案）；用偏好数据训 RM 把"好坏判断"变成可优化的标量奖励

    最容易误解

    • "RLHF 就是用人类标注的答案做 SFT"——错。RLHF 用的是偏好对比（A 比 B 好），不是标准答案
    • "RM 直接给出最佳答案"——错。RM 只打分，生成答案靠 policy 自己探索
    • "PPO 阶段不需要参考模型"——错。KL 约束需要参考模型作锚点，拉住 policy 不跑偏
    • "reward 越高模型越好"——错。reward 是 RM 给的近似值，reward hacking 时 reward 高但回答其实差

    来源：Christiano et al. 2017《Deep Reinforcement Learning from Human Preferences》；Stiennon et al. 2020《Learning to summarize from human feedback》；Ouyang et al. 2022《Training language models to follow instructions with human feedback》（InstructGPT）；Bai et al. 2022《Training a Helpful and Harmless Assistant》。PPO ← Schulman et al. 2017；BT 模型 ← Bradley & Terry 1952；策略梯度定理 ← Williams 1992 (REINFORCE)。

    前置：无。所有术语本页解释。想深入可看 backward（链式法则）和 DPO（RLHF 的简化替代）。

    小白版一句话定义：RLHF 是三步流程——先用 SFT 训基础模型，再用人类偏好对比训一个奖励模型 RM，最后把 RM 当环境用 PPO 训 policy，加 KL 约束防止 policy 跑偏。

    一、定位：它在整个流程里的哪一步

    LLM 训练的大流程是三段：预训练 → SFT → 对齐。RLHF 属于"对齐"层，目的是让模型行为符合人类偏好。

    具体定位：

    • 上游：预训练得到基础语言模型，SFT 让模型学会指令跟随。RLHF 从 SFT 模型起步，把这个 SFT 模型冻住作为参考模型 $\pi_{\text{ref}}$（参考模型：训练时不更新参数的基准模型，作为 KL 约束的锚点）。
    • 输入：偏好数据集 $\mathcal{D}_{\text{pref}}$，每条是一个偏好对（preference pair）$(x, y_w, y_l)$——一个 prompt $x$ 配两个答案，$y_w$ 是 chosen（被人类选中的），$y_l$ 是 rejected（被人类拒绝的）。
    • 下游：得到对齐后的 policy $\pi_\theta$，可以部署或继续做 DPO/RLHF 微调。

    本页里 policy（策略）就是指 LLM 本身——RL 文献里叫 policy，LLM 工程里叫模型，指的是同一个东西。$\pi_\theta$ 表示参数为 $\theta$ 的当前训练中的 policy。

    二、旧世界：没有它时会怎么做

    没有 RLHF 时，做完 SFT 就发布。SFT（监督微调）是模仿学习：给 prompt 配一个"标准答案" $y$，模型学最大似然——让 $\log \pi(y|x)$ 越大越好。

    $$\pi_{\text{SFT}} = \arg\max_\pi \, \mathbb{E}_{(x,y) \sim \mathcal{D}_{\text{SFT}}}[\log \pi(y|x)]$$

    符号说明：$\pi$ 是 policy（模型），$\mathcal{D}_{\text{SFT}}$ 是 SFT 示范数据集，$\mathbb{E}$ 是期望（在数据集上求平均），$\arg\max$ 表示"让目标最大的那个 $\pi$"。

    这个流程的实质是模仿：标注员从头写出"好的回答"作为示范数据，模型学训练数据里"回答长什么样"。它学的不是"什么回答是好的"，而是"训练分布里回答长什么样"。

    三、断点：为什么旧办法自然走不下去

    SFT 在开放任务上有 4 个走不下去的具体断点：

    1. 开放任务没有唯一标准答案。写诗、对话、总结有无数种好答案，SFT 只能让模型学其中一种，剩下的好答案都被压低概率。
    2. 人类擅长比较不擅长生成。标注员看两个答案能秒判哪个好，但让自己从头写一个好答案，质量上限低——标注员不是领域专家。
    3. 模仿 = 继承偏见。SFT 学训练数据，数据有什么偏见、有害内容、风格偏差，模型就学什么。没有机制说"这个回答虽然像训练数据但不好"。
    4. 偏好是相对的不是绝对的。人类偏好本质是"A 比 B 好"，不是"A 是对的"。SFT 把它硬塞成绝对形式（"A 是标准答案"），丢掉了比较结构。

    核心断点：人类能判断好坏，但难以把"好坏"直接写成示范答案。需要一个机制把人类的比较判断变成可优化的训练信号。

    四、新想法：这个概念改变了哪一步

    改变点：让人类当裁判而不是当老师。

    • 旧世界：人类当老师，从头写"标准答案" → 模型模仿答案
    • 新想法：人类当裁判，对比两个答案说哪个好 → 模型学"什么是好"

    具体做法分三步：

    1. 采偏好数据：让 SFT 模型对同一 prompt 采样几个回答，让标注员两两对比说哪个好，得到偏好对 $(x, y_w, y_l)$。
    2. 训 RM：用偏好对训一个奖励模型 RM——RM 学习给任意 (prompt, answer) 打一个标量 $r(x,y)$（标量 = 单个数字，区别于向量），分数高表示人类更喜欢。RM 把"人类偏好判断"变成可优化的数字。
    3. 用 RL 训 policy：把 RM 当环境，用 RL（强化学习）算法训 policy $\pi_\theta$ 去 maximize reward。policy 自己采样回答、RM 打分、policy 根据分数调整——这样能探索出比训练数据更好的回答。

    但这里有个新问题：如果只最大化 reward，policy 会钻 RM 的空子——找一些 RM 给高分但人类其实觉得烂的回答。这叫 reward hacking（奖励黑客，policy 骗 RM 拿高分但回答差）。

    解决方法：在目标里加 KL 约束——policy 不能离参考模型 $\pi_{\text{ref}}$ 太远。KL 散度（Kullback-Leibler divergence，衡量两个概率分布差异的指标，记作 $D_{\text{KL}}$）作为惩罚项：偏离越大，惩罚越重。这样 policy 在"拿高分"和"不离参考模型太远"之间找平衡，reward hacking 被缓解。

    读到这里，你应该已经能解释：

    • 为什么 SFT 在开放任务上走不下去？
    • RLHF 用什么数据替代了 SFT 的"标准答案"？
    • RM 在 RLHF 里扮演什么角色？
    • reward hacking 是什么，KL 约束怎么缓解它？

    五、正式定义：它到底是什么

    小白版定义：RLHF 是 3 步流程——SFT 起步、用偏好对训 RM、用 PPO 在 RM 当环境里训 policy，加 KL 约束防跑偏。

    正式技术定义：3 步流程，每步一个优化目标。

    第 1 步 SFT：用监督数据训基础模型。

    $$\pi_{\text{SFT}} = \arg\max_\pi \, \mathbb{E}_{(x,y) \sim \mathcal{D}_{\text{SFT}}}[\log \pi(y|x)]$$

    得到 $\pi_{\text{SFT}}$，作为后续 KL 约束的锚点 $\pi_{\text{ref}} = \pi_{\text{SFT}}$（冻结，训练时不更新参数）。

    第 2 步 训 RM：用偏好对训奖励模型。RM 是一个把 (prompt, answer) 映射到标量的网络 $r_\phi(x,y)$，参数 $\phi$。

    $$\max_{r_\phi} \, \mathbb{E}_{(x,y_w,y_l) \sim \mathcal{D}_{\text{pref}}}[\log \sigma(r_\phi(x,y_w) - r_\phi(x,y_l))]$$

    第 3 步 PPO 训 policy：用 RM 当环境，PPO 优化 policy。

    $$\max_{\pi_\theta} \, \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta}[r_\phi(x,y)] - \beta \, D_{\text{KL}}[\pi_\theta(\cdot|x) \| \pi_{\text{ref}}(\cdot|x)]$$

    符号说明（按出现顺序）：

    • $\pi$：policy，LLM 里就是模型本身
    • $\pi_\theta$：当前训练的 policy，参数 $\theta$ 可更新
    • $\pi_{\text{ref}}$：SFT 后冻结的参考模型，作为 KL 约束的锚点
    • $r_\phi(x,y)$：奖励模型 RM 给 (prompt $x$, answer $y$) 打的标量分数，参数 $\phi$
    • $y_w, y_l$：偏好对里 chosen / rejected 的答案
    • $\sigma(z) = 1/(1+e^{-z})$：sigmoid 函数，把任意实数压到 (0,1)，可解读为概率
    • $\mathcal{D}_{\text{SFT}}, \mathcal{D}_{\text{pref}}, \mathcal{D}$：SFT 示范数据集、偏好对比数据集、prompt 分布
    • $D_{\text{KL}}[P \| Q]$：KL 散度，$\sum_y P(y) \log(P(y)/Q(y))$，衡量分布 $P$ 相对 $Q$ 的差异
    • $\beta$：KL 惩罚强度（实践共识取 0.01~0.1 量级，来源：InstructGPT 等论文实践）。$\beta$ 大 → 偏离越多
    • $\mathbb{E}$：期望，对数据集求平均
    • $\arg\max$：让目标最大的那个参数

    六、机制：它怎么工作

    直觉层

    RLHF 让人类当裁判，模型自己探索怎么拿高分。RM 把"好坏判断"变成可优化的数字，PPO 把"拿高分"变成可优化的目标。三步串起来：先有基础模型 → 把人类偏好变成奖励信号 → 用奖励信号驱动 policy 探索。

    流程层

    步骤 1：SFT 起步。用监督数据 $\mathcal{D}_{\text{SFT}}$ 训基础模型，得到 $\pi_{\text{SFT}}$。这一步是 RLHF 的起点，也是后续 KL 约束的锚点——把 $\pi_{\text{SFT}}$ 冻结成 $\pi_{\text{ref}}$，训练时不更新。

    步骤 2：训 RM。对每个偏好对 $(x, y_w, y_l)$，让 RM 给两个答案打分 $r_\phi(x, y_w)$ 和 $r_\phi(x, y_l)$。用 BT 模型（Bradley-Terry，把"$y_w$ 比 $y_l$ 好"建模成概率的统计模型）把人类偏好变成概率：

    $$P(y_w \succ y_l | x) = \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

    RM 损失是最小化负对数似然：

    $$\mathcal{L}_{\text{RM}} = -\log \sigma(r_\phi(x, y_w) - r_\phi(x, y_l))$$

    最小化它就是让 RM 给 chosen 打分高于 rejected。RM 训好后，它能把任意 (prompt, answer) 映射到一个标量分数，分数高低反映人类偏好。

    步骤 3：PPO 训 policy。流程是：

    1. policy $\pi_\theta$ 在 prompt $x$ 下采样回答 $y \sim \pi_\theta(\cdot | x)$（on-policy，用当前 policy 采的数据训当前 policy）。
    2. RM 给 $y$ 打分 $r_\phi(x, y)$，作为 reward。
    3. 用 PPO（Proximal Policy Optimization，一种策略梯度 RL 算法，核心是 clip 限制每次更新幅度防破坏训练稳定性）算策略梯度：reward 高的回答概率上升，低的下降。
    4. 算 KL 惩罚 $\beta \, D_{\text{KL}}[\pi_\theta(\cdot|x) \| \pi_{\text{ref}}(\cdot|x)]$，从 reward 里减掉。
    5. 更新 $\theta$，回到步骤 1 继续采。

    完整目标：$\max_{\pi_\theta} \mathbb{E}[r_\phi(x,y)] - \beta D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}]$。第一项推 policy 拿高分，第二项拉 policy 不离参考模型太远。

    展开：BT 模型怎么来的、为什么用 sigmoid

    BT 模型（Bradley-Terry）来自 Bradley & Terry 1952，是个经典的偏好建模框架。核心假设：每个选项 $y$ 有一个"实力值" $r(y)$，选项 A 比 B 好的概率是 $\sigma(r(A) - r(B))$。

    这个形式不是凑出来的，是从"两个选项的实力差应该决定胜负概率"这个要求出发，唯一满足一组合理公理（如对称性、传递性）的函数形式就是 sigmoid of difference。

    在 RLHF 里，RM 输出的标量 $r_\phi(x,y)$ 就是这个"实力值"——分数越高表示人类越喜欢。把偏好对代入 BT 模型，就得到 $\mathcal{L}_{\text{RM}}$。

    来源：BT 模型 ← Bradley & Terry 1952；RLHF 用 BT 训 RM ← Christiano et al. 2017。

    展开：PPO 目标怎么变可优化（策略梯度 + 优势函数 + clip）

    RLHF 第 3 步的目标 $\mathbb{E}_{y \sim \pi_\theta}[r_\phi(x,y)] - \beta D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}]$ 看起来抽象，怎么变成可算梯度的更新？

    第 1 步：展开 KL 项。$D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}] = \mathbb{E}_{y \sim \pi_\theta}[\log(\pi_\theta(y|x) / \pi_{\text{ref}}(y|x))]$。代入目标：

    $$\mathbb{E}_{y \sim \pi_\theta}\left[ r_\phi(x,y) - \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)} \right]$$

    这等价于把"偏离参考模型"作为软惩罚加到每条采样上。

    第 2 步：用策略梯度。对 $\mathbb{E}_{y \sim \pi_\theta}[f(y)]$ 求 $\theta$ 梯度，标准结果是 $\mathbb{E}_{y \sim \pi_\theta}[f(y) \nabla_\theta \log \pi_\theta(y|x)]$（策略梯度定理，REINFORCE）。

    这里 $f(y) = r_\phi(x,y) - \beta \log(\pi_\theta(y|x) / \pi_{\text{ref}}(y|x))$。但直接用 reward 当 $f$ 方差大，所以减一个 baseline $V(x)$（value function，价值函数，估计 prompt $x$ 下的平均 reward）。

    定义优势函数（advantage function）$A(x,y) = r_\phi(x,y) - V(x)$——某条回答相对平均好多少。梯度变成：

    $$\nabla_\theta J \approx \mathbb{E}_{y \sim \pi_\theta}[A(x,y) \, \nabla_\theta \log \pi_\theta(y|x)] - \beta \nabla_\theta D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}]$$

    第 3 步：PPO 的 clip。直接用上面梯度更新，步子大了训练会崩。PPO 用重要性比 $r_t(\theta) = \pi_\theta(y|x) / \pi_{\theta_{\text{old}}}(y|x)$（新 policy 给 $y$ 的概率除以采时 policy 给的概率），clip 到 $[1-\epsilon, 1+\epsilon]$：

    $$\mathcal{L}^{\text{CLIP}}(\theta) = \mathbb{E}[\min(r_t A, \, \text{clip}(r_t, 1-\epsilon, 1+\epsilon) \, A)]$$

    $\epsilon$ 是 clip 范围（实践共识 0.1~0.2 量级）。clip 限制每次更新幅度，避免 policy 一次跳太远破坏训练稳定性。

    本节符号说明：$V(x)$ 是 value function（baseline），$\theta_{\text{old}}$ 是采数据时的 policy 参数，$r_t$ 是新旧 policy 概率比，$A$ 是优势函数，$\epsilon$ 是 clip 范围。

    来源：策略梯度定理 ← Williams 1992 (REINFORCE)；PPO ← Schulman et al. 2017。

    展开：KL 约束为什么能缓解 reward hacking

    reward hacking 的根因：RM 是个有界的近似模型，它在训练分布外（OOD, out-of-distribution）的打分不可信。policy 一旦探索到 OOD 区域，RM 给的高分可能并不对应人类真实偏好——policy 就会往这些"虚假高分"区域跑。

    KL 约束的作用：$\beta D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}]$ 拉住 policy 不离 $\pi_{\text{ref}}$ 太远。$\pi_{\text{ref}}$ 是 SFT 模型，它的分布是"在训练数据上学到的合理回答分布"——RM 在这个分布附近打分是可信的（因为偏好数据就采自这里）。

    所以 KL 约束的实质是：把 policy 限制在 RM 打分可信的区域内。偏离越远，惩罚越大，policy 就不敢跑到 OOD 去钻空子。

    $\beta$ 越大约束越强——policy 更保守但 reward 上限也低；$\beta$ 越小约束越弱——policy 更激进但更容易 reward hacking。实践共识 $\beta$ 在 0.01~0.1 量级（来源：InstructGPT 等论文实践）。

    展开：最小数值例子（RM loss + PPO 一步更新）

    设定（本文构造，纯教学用）：词表只有 3 个 token，prompt 是 token 0，可能回答是 token 1 / 2 / 3。偏好对里 chosen = token 1, rejected = token 2。

    RM 训练一步：

    设 RM 当前给三个回答的分数 $r_\phi = [1.5, 0.2, 0.3]$（对应 token 1/2/3）。chosen 是 token 1（$r=1.5$），rejected 是 token 2（$r=0.2$）。

    BT 概率：$\sigma(r_{\text{chosen}} - r_{\text{rejected}}) = \sigma(1.5 - 0.2) = \sigma(1.3)$。

    $\sigma(1.3) = 1/(1+e^{-1.3}) = 1/(1+0.2725) = 1/1.2725 \approx 0.786$。

    RM loss = $-\log 0.786 \approx 0.241$。

    梯度方向：让 $r_{\text{chosen}}$ 上升、$r_{\text{rejected}}$ 下降，loss 下降。

    PPO 训练一步（本文构造）：

    设参考模型 $\pi_{\text{ref}}$ 在 prompt 下对三个 token 的概率 $= [0.6, 0.3, 0.1]$。

    当前 policy $\pi_\theta$ 的概率 $= [0.55, 0.35, 0.10]$（已经往 chosen 方向偏了一点）。

    采样得到 token 1（chosen），RM 给分 $r_\phi(x, \text{token 1}) = 1.5$。

    设 baseline $V(x) = 0.5$（本文构造），优势 $A = 1.5 - 0.5 = 1.0$。

    重要性比 $r_t = \pi_\theta(\text{token 1}|x) / \pi_{\theta_{\text{old}}}(\text{token 1}|x)$。设 $\theta_{\text{old}}$ 时 $\pi = [0.6, 0.3, 0.1]$（和 ref 一样），则 $r_t = 0.35 / 0.3 \approx 1.167$。

    设 $\epsilon = 0.2$，clip 范围 $[0.8, 1.2]$，$r_t = 1.167$ 在范围内不 clip。

    clip 项 = $\min(r_t \cdot A, \, \text{clip}(r_t, 0.8, 1.2) \cdot A) = \min(1.167 \times 1.0, 1.167 \times 1.0) = 1.167$。

    KL 项：$D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}] = 0.55\log(0.55/0.6) + 0.35\log(0.35/0.3) + 0.10\log(0.10/0.10)$

    $= 0.55 \times \log 0.917 + 0.35 \times \log 1.167 + 0$

    $= 0.55 \times (-0.0866) + 0.35 \times 0.1544$

    $= -0.0476 + 0.0540 = 0.0064$。

    设 $\beta = 0.1$，KL 惩罚 = $0.1 \times 0.0064 = 0.00064$。

    这一步的目标 $\approx 1.167 - 0.00064 = 1.166$。policy 沿这个目标的梯度方向更新 $\theta$，让 chosen 概率继续上升。

    关键观察：KL 项很小（0.00064），是因为 $\pi_\theta$ 离 $\pi_{\text{ref}}$ 还很近。如果 $\pi_\theta$ 跑到 $[0.1, 0.8, 0.1]$（偏离很远），KL 会变大很多，惩罚就显著——这就是 KL 约束"拉住"的机制。

    所有数字均为本文构造，教学用。

    展开：RLHF 三步伪代码 
    # RLHF 三步流程伪代码（教学用，省略工程细节）

# ===== 第 1 步：SFT =====
# 用监督数据 (x, y) 训基础模型
for (x, y) in sft_dataset:
    loss = -log_prob(model, y | x)   # 最大似然 = 最小化负对数似然
    loss.backward()
    optimizer.step()
pi_ref = copy(model)                 # 冻结成参考模型
pi_ref.eval()                        # 后续只读不更新

# ===== 第 2 步：训 RM =====
# 用偏好对 (x, y_w, y_l) 训奖励模型
rm = RewardModel()                   # 把 (x, y) 映射到标量
for (x, y_w, y_l) in pref_dataset:
    r_w = rm(x, y_w)                 # chosen 的分数
    r_l = rm(x, y_l)                 # rejected 的分数
    loss = -log_sigmoid(r_w - r_l)   # BT 模型的负对数似然
    loss.backward()
    optimizer.step()

# ===== 第 3 步：PPO 训 policy =====
policy = copy(pi_ref)                # 从 SFT 起步
for x in prompt_dataset:
    y = policy.sample(x)             # on-policy 采样
    r = rm(x, y)                     # RM 打分
    A = r - value_fn(x)              # 优势 = reward - baseline
    ratio = policy.prob(y | x) / old_policy.prob(y | x)
    clipped = clamp(ratio, 1 - eps, 1 + eps)
    ppo_loss = -min(ratio * A, clipped * A)   # PPO clip 目标
    kl = kl_div(policy(x), pi_ref(x))
    total_loss = ppo_loss + beta * kl
    total_loss.backward()
    optimizer.step()

    七、边界：什么时候用，什么时候不用

    RLHF 适合开放性任务——需要符合人类偏好但难以定义标准答案的场景。它不解决一切：

    • 有明确标准答案的任务（数学、代码）用 RLVR（Reinforcement Learning with Verifiable Rewards，可验证奖励 RL，用规则或程序直接判定对错作为 reward）更合适——不需要训 RM，reward 完全可信。
    • 偏好数据获取成本高的场景考虑 RLAIF（RL from AI Feedback，用 AI 反馈代替人类反馈）——便宜但质量受限于 AI 评判能力。
    • 计算预算有限用 DPO（直接偏好优化，跳过 RM 和 PPO，一个 loss 在偏好数据上训 policy）——更便宜更稳定，但不能持续探索。
    方法解决什么不解决什么什么时候用
    RLHF（本文）开放任务的偏好对齐，能持续探索新回答训 RM 贵、PPO 不稳定、reward hacking开放任务 + 有预算训 RM 调 PPO
    SFT快速让模型学会指令跟随学不到偏好，受示范数据上限预算有限、任务有标准答案
    DPO跳过 RM 和 PPO，直接在偏好数据上训 policy不能持续探索、受偏好数据覆盖范围限制偏好数据充足 + 想要训练稳定
    RLVR用可验证规则当 reward，reward 完全可信只适合有客观对错的任务数学、代码、推理任务
    RLAIF用 AI 反馈代替人类反馈，便宜AI 评判能力有限、可能有偏见偏好数据采集成本高
