论文原文

MemAgent：用"固定长度记忆 + RL"把长上下文 LLM 变成线性复杂度的循环网络

给非专业读者的 3 分钟导读。技术细节看 主文档，论文原文看 paper.pdf。

这个领域原来有什么问题

想象你要读 1000 页资料回答一个跨多页的问题。当前 LLM 的做法是把所有内容一次性塞进"工作记忆"——但工作记忆有上限，塞不下就只能截断；更糟的是注意力机制的计算成本是平方级，上下文长 100 倍、算力要 1 万倍。

更微妙的是，即便模型标称支持长上下文，真正用到极限时性能也会塌方。比如标称支持 1M token 的 Qwen2.5-Instruct-1M，在 896K 上准确率已经掉到 0%——还没到 1M 就崩了。业界把"长度变长就掉性能"称为外推失败。已有的三大方案——长度外推（移位置编码 + 继续预训练）、稀疏 / 线性注意力、上下文压缩——各有短板，没有方法能同时做到"无限长度 + 无损外推 + 线性复杂度"这三件事。

本文做了什么

MemAgent 让 LLM 像人读长文档做笔记：把长文档切成一段段，每读一段就覆盖更新一份固定长度的"记忆"（1024 token 的笔记本），全部读完只看记忆和问题答出最终答案。记忆就是普通文本 token，模型架构完全没动。

但"光有记忆机制不够"——如果模型不知道什么该记、什么该扔，性能仍会随长度衰减。论文的关键创新是 Multi-Conv DAPO 算法：把每段记忆更新当成一次独立的 RL 对话，用规则验证器（答案对不对）作为奖励信号，端到端训练模型学会"做笔记"。每个 query 拆成 5-7 个上下文独立的对话分别优化，最后那个含答案的对话算奖励，分摊给前面所有对话。这绕开了反向传播无法穿过"覆盖"这种离散操作的瓶颈。

效果如何

训练只在 8K 上下文窗口 + 32K 长度的 HotpotQA 衍生数据上做，测试时外推到 3.5M token（437 倍）。完整对比（准确率 %）：

3 个关键数字：

  • 437 倍外推仍保持 78% 准确率：14B 模型从 7K（83.59%）到 3.5M（78.12%）仅下降 5.47 个百分点
  • OOD 任务 95%+：训练只见过 HotpotQA 衍生的多跳 QA，在 RULER 的 NIAH / 变量追踪 / 频繁词 / SQuAD QA 上 8K-512K 平均 > 95%
  • 复杂度从 O(n²) 降到 O(n)：3.5M token 上基线模型算力爆炸，MemAgent 线性增长

所有基线在 896K 都已塌方（标称 1M 的模型 0%，128K 推理模型 7-12%），而 MemAgent 在 3.5M 仍保持 78%——这是端到端 RL 训练"做笔记"能力的结果，不是靠扩大上下文窗口。消融实验进一步证明：光加记忆机制不训 RL，性能仍随长度衰减；只有 RL 训练的记忆才是关键。

一句话启示

长上下文的瓶颈不在"能不能装下"，而在"能不能学会选择性记忆"——MemAgent 用 RL 直接优化"做笔记"这个动作本身，绕开了人工稀疏模式或特征空间压缩。

局限：训练数据只用 HotpotQA 衍生的多跳 QA（OOD 评测覆盖但训练单一）；记忆固定 1024 token 对极复杂多跳推理可能是瓶颈；覆盖策略是破坏性的，信息被覆盖后无回查原文机制（与 RAG 不同）；推理虽是 $O(N)$ 但需 5-7 轮串行对话增加延迟；评测主要在英文 QA 类任务，多语言和开放式生成未评估。
