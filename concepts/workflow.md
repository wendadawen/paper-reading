# 概念教学文档写作流程

**产物**：纯 HTML + KaTeX（CDN）。每个概念一个目录，主入口 `index.html`。

**读者画像**：零基础入门训练/RL/post-training 的工程师。有技术基础但不懂这个领域。30 秒能抓要点，展开折叠能看到完整公式推导。

**模板**：`concepts/template/index.html`，拷贝改占位符即可，不要从空白 HTML 写起。模板已实现默认结构、样式、暗/亮模式、TOC、进度条、返回顶部、KaTeX auto-render。

**与论文解读的差异**：概念教学要完整公式推导（展开 `<details>` 能看到每一步数学变换），论文解读只要最终式。详见 `papers/workflow.md`。

---

# 一、流程

### 1. 确认概念范围

回答三个问题，答案贯穿全文：
1. **一句话本质**：这个概念用 X 解决了 Y 问题
2. **痛点是什么**：为什么需要这个概念（前置概念的什么局限）
3. **核心公式**：哪几个公式必须推导清楚

### 2. 拷贝模板

```bash
mkdir -p concepts/<concept-name>
cp concepts/template/index.html concepts/<concept-name>/index.html
```

改占位符：`<title>`、`<h1 class="title">`、nav-brand（写具体概念名，如"RLHF"）、blockquote.meta 里的来源和前置。

### 3. 写内容

按"规范"部分的结构写。公式推导放 `<details>` 折叠——展开要能看到完整推导链条，每一步数学变换都写出来，默认读者是小白。

### 4. 核查

按"核查清单"逐项实查。外行友好度必须模拟读者实际读一遍，不能只跑 grep。

# 二、规范

## 主文档结构

严格按此顺序，H1 用结论式（"一、问题：SFT 教不会模型谁比谁好"）而非主题式（"一、问题"）：

```
顶部 callout（30 秒层）
  问题 → 方案 → 产物，各一句话。不写"效果"（概念教学没有单一实验数据）。

blockquote.meta
  来源（哪篇论文/哪本书/哪个领域常识）+ 前置（学这个前要先懂什么）。

"这个概念一句话"段
  一段话展开成 2-3 句。callout 和正文的过渡。

术语速查表
  只列正文反复出现的术语。每个用日常语言解释（不能用未解释的术语解释术语）。

一、问题（H1，结论式标题）
  本章你将知道：<一句话>
  先讲痛点，再引出这个概念要解决什么。

二、方法（H1，结论式标题）
  本章你将知道：<一句话>
  先讲直觉（一段话，不贯穿全文），再给公式。
  公式推导放 <details> 折叠——展开要能看到完整推导链条：
    目标 → 中间步骤 → 最终式。每一步数学变换都写出来。
  代码片段（训练伪代码、工程实现）也放 <details class="code-details"> 折叠——
    和公式推导同样的理由：默认收起，感兴趣的读者展开看。
    里面用 <div class="code-block"><pre><code>...</code></pre></div> 包裹，保留复制按钮。

三、总结 / 对比（H1，结论式标题）
  本章你将知道：<一句话>
  方法本质回顾 + 和相关概念的对比（用表格）。

证据边界声明（H1）
  本章你将知道：每个论断的出处。
  分两段：事实/公式出处 + 解读者推断。
```

每个 H1（含证据边界声明）下面都要有"本章你将知道"。

## 笔墨分配

| 章节 | 占全文比例 | 超标怎么办 |
|------|----------|----------|
| 问题 | 15-20% | 砍背景细节，只留痛点 |
| 方法 | 40-50% | 推导放折叠块，正文只讲直觉和最终式 |
| 总结/对比 | 10-15% | 不超过 200 字 |
| 术语 + 证据声明 | 15-20% | — |

方法章节内部：直觉 30% / 公式推导（折叠）50% / 性质与好处 20%。

## 写作要点

- 顶部 callout 不写"效果"——概念教学没有单一实验数据。写"产物"（学完能理解什么）。
- H1 结论式标题，串起来能看出概念的故事线。
- 公式默认折叠（`<details>`），展开是完整推导不是只有最终式。
- 代码片段（训练伪代码、工程实现）也默认折叠（`<details class="code-details">`），和公式推导同样理由：默认收起，感兴趣的读者展开看。
- 关键假设用 callout 在主路径标注，不藏折叠块。
- 类比克制使用：作为辅助理解的工具用一句话点一下，不贯穿全文，不把整个文档包装成故事。
- 数值例子用真实场景（prompt + 回答），不用抽象 token。
- 导航栏 nav-brand 写"概念学习"，不要写"方法名：一句话概括"。
- 不要残留模板占位符（summary.html / paper.pdf 链接等）——概念教学没有这些。
- 证据出处集中在"证据边界声明"H1 下，不散在正文里。

# 三、核查清单

写完浏览器打开 HTML，逐项实查。**外行友好度必须模拟读者实际读一遍。**

### 外行友好度（最重要）

- [ ] 只读 callout + 各 H1 标题 + "本章你将知道"，能不能向别人复述这个概念做了什么？
- [ ] 术语表里有没有用未解释的术语解释术语？换成日常语言
- [ ] 方法章节第一段是不是直觉？如果是公式或推导，重写
- [ ] 找一句最技术的话，想象讲给不懂这个领域的朋友听——他能不能懂？

### 公式推导完整（概念教学特有）

- [ ] 每个关键公式都有 `<details>` 折叠推导？
- [ ] 展开后是完整推导链条，不是只有最终式 + 变量解释？
- [ ] 每一步数学变换都解释了为什么（如 log 技巧、指数换元、为什么取负）？
- [ ] 默认读者是小白——数学工具（sigmoid、log、期望、梯度）第一次出现时解释？
- [ ] 代码片段（训练伪代码、工程实现）也用 `<details class="code-details">` 折叠？

### 事实准确

- [ ] 每个论断有出处，集中在"证据边界声明"H1 下
- [ ] 关键假设在主路径 callout 标注，不只藏在折叠块
- [ ] 证据边界声明区分"事实/公式出处"和"解读者推断"

### 格式

- [ ] H1 都是结论式，串起来能看出概念故事线
- [ ] 每个 H1 下都有"本章你将知道"
- [ ] KaTeX 无红色 ParseError；行内数学符号都用 `$...$` 包裹
- [ ] 没有 `<code>$...$</code>`（KaTeX auto-render 不扫描 `<code>` 内部）
- [ ] callout 5-10 个，颜色语义统一（蓝=提示/绿=正向/红=警告/黄=注意/紫=重点/灰=备注）
- [ ] 对比数据用表格
- [ ] 每章都有视觉锚点（图/表/callout 至少一个），无连续两屏纯文字
- [ ] 没有模板占位符残留（"方法名：一句话概括"、summary.html 链接等）

# 四、铁律（违反=文档不可用）

1. **公式推导完整**：每个关键公式都要有完整推导（`<details>` 折叠展开），每一步数学变换都写出来，默认读者是小白。概念教学的核心价值就是让读者真正理解方法怎么来的。

2. **直觉先行**：方法章节先讲直觉再给公式。判断标准：把所有公式块删掉，方法章节应该还能让人懂方法在做什么。

3. **关键假设不藏折叠块**：方法章节的任何假设必须在主路径 callout 标注。读者把"假设下的结论"当成"无条件的事实"是文档的失败。

4. **不残留模板占位符**：nav-brand、title、blockquote.meta 等都要改成实际内容。概念教学没有 summary.html / paper.pdf 链接。

5. **证据出处集中标注**：每个论断有出处，集中在"证据边界声明"H1 下。找不到出处就标"解读者推断"。

## 数学渲染

- 变量和公式用 LaTeX，不要混工程变量名。
- JS 字符串里 LaTeX 命令写双反斜杠：`\\frac`、`\\partial`。
- 不用 `\mathbb{1}`，用 `\delta_{ky}`。
- 不用 `\log(数字)`，KaTeX 会把括号当函数参数。用 `\log p = 数字`。
- 不要把 `$...$` 包在 `<code>` 标签里——KaTeX auto-render 不扫描 `<code>` 内部。
- 数学推导默认零基础：出现的符号都要在术语速查或公式后解释。

## 常见 bug

| 现象 | 原因 | 解决 |
|---|---|---|
| `$...$` 原样显示 | KaTeX 没加载 | 检查 CDN |
| `<code>$...$</code>` 不渲染 | KaTeX auto-render 不扫描 `<code>` 内部 | 去掉 `<code>`，直接用 `$...$` |
| `rac{...}` 乱码 | JS 把 `\f` 当 form feed | 写 `\\frac` |
| `k=y` 显示异常 | `\mathbb{1}` 不支持 | 用 `\delta_{ky}` |
| `\neq` 渲染成方块 | 字体没加载完整 | 检查 KaTeX CSS/fonts |
| `\log(0.5)` 渲染异常 | 括号被当函数参数 | 用 `\log p = 数字` |
| 公式部分渲染部分不渲染 | 批量替换破坏字符串 | 手动逐处改 |
| 中间结果被画成"也被更新" | 混淆参数和中间结果 | 更新公式只作用于参数，中间结果用"重算"描述 |
| 数值凭空出现 | 跳步 | 每个数字给来源 |
| 页面出现订正口吻 | 把纠错过程当教学内容 | 删掉，只讲正确关系 |
| 暗/亮模式切换无效 | CSS `@media` 和 JS `data-theme` 不一致 | 用 `html[data-theme="light/dark"]` 显式定义变量，`@media` 只处理 `html:not([data-theme])` |
