# 概念教学文档写作规范

## 第一准则：正确性

教学文档的错误会直接误导读者，比不写更糟。正确性高于美观、高于完整性、高于进度。

写完每一页后必须做两步审查：

1. 逐页搜索审查：每一页出现的公式、概念、数值，都要去原始来源（论文、官方文档、教科书）核对，不能凭记忆写。DPO 的 loss 是不是论文公式 7？softmax 的导数是不是写反了？梯度符号对不对？每个论断都要有出处。
2. 逐页数值审查：用 Python 独立复算每一页显示的数值，从第一步输入手算到最后一步结果，确认链条一致、关键不变式成立（训练前 logRatio = 0、loss 单调下降、梯度方向正确等）。不能凭"数字看起来对"就放过。

审查不通过就不能发布。宁可少写一页，不可错写一页。

## 目录结构

每个概念一个目录，主入口叫 `index.html` 或 `index.md`。

```text
concepts/
├── workflow.md
├── training-landscape.md
├── backward/index.html
├── dpo/index.html
└── rlhf/index.html
```

新增：`concepts/<name>/index.html`

## HTML 依赖

- 单 HTML 文件，CSS / JS 内联。
- KaTeX 用 CDN，和 `papers/` 保持一致，不单独放 `assets/`。
- 推荐：直接拷贝 `papers/template/index.html` 作为骨架（含暗/亮模式、TOC、进度条、返回顶部、KaTeX auto-render 等全套功能），改占位符即可，不要从空白 HTML 写起。
- KaTeX 用 auto-render（模板已配置），行内 `$...$`、块级 `$$...$$`。

```html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.css">
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.9/dist/katex.min.js"></script>
<script defer src="https://cdn.jsdelivr.net/npm/katex@0.16.9/contrib/auto-render.min.js"
  onload="renderMathInElement(document.body, {
    delimiters: [
      {left: '$$', right: '$$', display: true},
      {left: '$', right: '$', display: false}
    ],
    throwOnError: false
  });"></script>
```

## 概念教学 vs 论文解读的深度差异

- 概念教学（`concepts/` 下）：要完整公式推导。读者是为了真正理解方法，需要知道"怎么来的、具体怎么做的、公式怎么推导的"。推导放 `<details>` 折叠，默认收起，读者展开能看到完整来龙去脉——每一步数学变换都写出来，默认读者是小白。
- 论文解读（`papers/` 下）：读者只是为了大概知道这篇论文做了什么。按 `papers/workflow.md` 的"公式推导按需展开——默认只给最终式"即可。

## 概念教学的结构

套用 `papers/template/index.html` 模板，按以下结构写（不是论文解读，是概念教学）：

```
顶部 callout（30 秒层）
  问题 → 方案 → 产物，各一句话。
  不写"效果"（概念教学没有单一实验数据可引）。

术语速查表
  只列正文反复出现的术语。每个用日常语言解释（不能用未解释的术语解释术语）。

一、问题（H1，结论式标题）
  本章你将知道：<一句话>
  先讲痛点，再引出这个概念要解决什么。

二、方法（H1，结论式标题）
  本章你将知道：<一句话>
  先讲直觉（一句话点一下，不贯穿全文），再给公式。
  公式推导放 <details> 折叠——展开要能看到完整推导链条：
    动机 → 中间步骤 → 最终式。每一步数学变换都写出来。

三、总结 / 对比（H1，结论式标题）
  本章你将知道：<一句话>
  方法本质回顾 + 和相关概念的对比（用表格）。

证据边界声明（H1）
  本章你将知道：每个论断的出处。
  分两段：论文/领域事实 + 解读者推断。
```

## 笔墨分配

| 章节 | 占全文比例 | 超标怎么办 |
|------|----------|----------|
| 问题 | 15-20% | 砍背景细节，只留痛点 |
| 方法 | 40-50% | 推导放折叠块，正文只讲直觉和最终式 |
| 总结/对比 | 10-15% | 不超过 200 字 |
| 术语 + 证据声明 | 15-20% | — |

方法章节内部：直觉 30% / 公式推导（折叠）50% / 性质与好处 20%。

## 写作要点

- 顶部 callout 不写"效果"——概念教学没有单一实验数据。写"产物"（这个方法产出什么）。
- H1 结论式标题，串起来能看出概念的故事线。
- 每个 H1 下都要有"本章你将知道"。
- 公式默认折叠（`<details>`），展开是完整推导不是只有最终式。
- 关键假设用 callout 在主路径标注，不藏折叠块。
- 类比克制使用：作为辅助理解的工具用一句话点一下，不贯穿全文，不把整个文档包装成故事。
- 数值例子用真实场景（prompt + 回答），不用抽象 token（hello/hi/there/bye）。
- 导航栏（nav-brand）写"概念学习"或概念名，不要写"方法名：一句话概括"。
- 不要残留模板占位符（summary.html / paper.pdf 链接等）——概念教学没有这些。
- 证据出处集中在"证据边界声明"H1 下，不散在正文里。

## 数学渲染

- 变量和公式用 LaTeX，不要混工程变量名。
- JS 字符串里 LaTeX 命令写双反斜杠：`\\frac`、`\\partial`。
- 不用 `\mathbb{1}`，用 `\delta_{ky}`。
- 箭头用 `\to` 或 `\rightarrow`。
- 不用 `\log(数字)`，KaTeX 会把括号当函数参数。用 `\log p = 数字`。
- 含 `$...$` 的字符串不能用 `innerHTML` 塞 DOM（如果自己控制渲染）。模板用 auto-render 则无此限制。
- 先挂 DOM 再调渲染函数（如果自己控制渲染）。
- 数学推导默认零基础：出现的符号（$\nabla$ $\partial$ $\sum$ $\delta_{ky}$ $\sigma$ $e$ $\log$ $\to$ 等）都要在术语速查或公式后解释。
- 不要把 `$...$` 包在 `<code>` 标签里——KaTeX auto-render 不扫描 `<code>` 内部，会显示成原样文本。

## 内容规范

- 先解释"这一步在干什么"，再给公式。公式不替换成人话，旁边加解释。
- 公式推导放 `<details>` 折叠——展开要能看到完整推导链条，每一步数学变换都写出来，默认读者是小白。
- 分清参数和中间结果：参数（权重、偏置等）是梯度下降更新的对象；中间结果（logits、概率、loss 等）是前向算出来的，参数变了重新前向才算出新的中间结果。更新公式只作用于参数，中间结果不写"也被更新"。
- 页面只写内容，不写写作元说明（"本文只讲 X""所有公式用 KaTeX 渲染"）。也不写订正过程（"关键区别""不要画成 X"）。

## 交互

- 用 papers 模板的长文档形式（可滚动），不是翻页动画。
- 保留模板的：暗/亮模式切换、侧边 TOC、进度条、返回顶部、图片点击放大。
- 公式推导用 `<details>` 折叠，默认收起。

## 视觉

- 继承 papers 模板的样式（暗/亮模式、serif 正文、distill 风格表格）。
- callout 颜色语义：蓝=提示/绿=正向/红=警告/黄=注意/紫=重点/灰=备注。
- 对比数据用表格，关键数字加粗。

## 流程

1. 拷贝 `papers/template/index.html` 作为骨架。
2. 改 `<title>`、`<h1 class="title">`、nav-brand、去掉 summary.html / paper.pdf 链接。
3. 按上面的结构写内容：callout → 术语速查 → 一、问题 → 二、方法 → 三、总结 → 证据边界声明。
4. 公式推导放 `<details>` 折叠，每一步数学变换都写出来。
5. 写完检查：没有模板占位符残留（"方法名：一句话概括"等）、没有 `<code>$...</code>`、JS 语法正常。
6. 按第一准则做逐页搜索审查 + 逐页数值审查（如果文档里有数值例子）。

## 常见 bug


| 现象               | 原因                              | 解决                                |
| ---------------- | ------------------------------- | --------------------------------- |
| `$...$` 原样显示     | KaTeX 没加载                       | 检查 CDN                            |
| 部分字段 `$...$` 不渲染 | 走了 `innerHTML` 没进 `renderMixed` | 用 `renderMixed` 或 `scanAndRender` |
| `<code>$...$</code>` 不渲染 | KaTeX auto-render 不扫描 `<code>` 内部 | 去掉 `<code>`，直接用 `$...$`。变量和公式都不要包在 `<code>` 里 |
| `rac{...}` 乱码    | JS 把 `\f` 当 form feed           | 写 `\\frac`                        |
| `k=y` 显示异常       | `\mathbb{1}` 不支持                | 用 `\delta_{ky}`                   |
| `\neq` 渲染成方块     | 字体没加载完整                         | 检查 KaTeX CSS/fonts                |
| `\log(0.5)` 渲染异常 | 括号被当函数参数                        | 用 `\log p = 数字`                   |
| 公式部分渲染部分不渲染      | 批量替换破坏字符串                       | 手动逐处改                             |
| 中间结果被画成"也被更新"    | 混淆参数和中间结果                       | 更新公式只作用于参数，中间结果用"重算"描述            |
| 数值凭空出现           | 跳步                              | 每个数字给来源                           |
| 页面出现订正口吻         | 把纠错过程当教学内容                      | 删掉，只讲正确关系                         |
