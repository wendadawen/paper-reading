    DPO：直接偏好优化

    问题：RLHF 要训 RM + 用 PPO 训 policy，3 步流程复杂、训练不稳定、多步误差累积。能否跳过 RM 和 RL，直接在偏好数据上训 policy？
    方案：DPO 从数学上证明：最优 policy 可以直接从偏好数据 + 参考模型算出来，不需要显式训 RM，也不需要 PPO。一个 loss 搞定
    产物：理解 DPO loss 公式怎么来的、为什么能替代 RLHF、什么场景用 DPO 什么场景用 RLHF

    来源：Rafailov et al. 2023《Direct Preference Optimization: Your Language Model is Secretly a Reward Model》（NeurIPS 2023）

    前置：RLHF 流程、softmax、sigmoid、对数概率、链式法则。不会的先看 RLHF 和 backward。

    DPO 一句话：它的核心洞察是 RLHF 的最优解有解析形式——把"最优 policy 相对参考模型的变化量"当成隐含奖励，代入 Bradley-Terry 偏好模型就能得到 DPO loss。这样跳过显式训 RM，直接在偏好数据上算 loss 训 policy。

    术语速查

    术语大白话
    偏好对$(x, y_w, y_l)$——一个 prompt 配两个答案，标注哪个好。$y_w$ 是 chosen，$y_l$ 是 rejected

    隐含奖励 $\hat{r}_\theta$$\beta \log(\pi_\theta(y|x) / \pi_{\text{ref}}(y|x))$——当前模型相对参考模型给某回答的概率变化。DPO 的核心量

    一、问题：RLHF 3 步太复杂，能否直接训 policy

    本章你将知道：RLHF 的痛点，DPO 要解决什么。

    RLHF 完整流程 3 步：偏好数据 → 训 RM → 用 PPO 训 policy。这个流程有几个痛点：

    1. 训 RM 是另一个模型训练：要单独训一个奖励模型，成本高，且 RM 质量直接影响后续训练
    2. PPO 是 RL 算法，训练不稳定：reward hacking（policy 骗 RM 给高分但回答差）、训练振荡、超参难调
    3. 多步流程误差累积：RM 训不准 → PPO 用错的 reward 信号 → policy 学错

    DPO 的核心问题：能不能跳过 RM 和 PPO，直接在偏好数据上训 policy？

    关键洞察：数学上可以证明，RLHF 的最优 policy 有一个解析形式（closed-form）。把这个最优解反解，能直接写出 reward 和 policy 的关系——把 reward 用 policy 表达出来，代入 BT 偏好模型，就得到一个直接在偏好数据上算的 loss。这就是 DPO。

    二、方法：从 RLHF 目标推出 DPO loss

    本章你将知道：DPO loss 公式怎么从 RLHF 目标推导出来，每步在干什么。

    直觉

    DPO 的推导思路是：RLHF 的目标函数有一个最优解，把最优解反解就能得到 reward 用 policy 表达的公式。

    具体来说，RLHF 目标是"最大化 reward，但不要离参考模型太远"（KL 约束）。这个目标有解析最优解——最优 policy 等于参考模型乘以 reward 的指数，再归一化。

    把这个最优解反过来用 policy 表达 reward：reward 等于 $\beta$ 乘以当前 policy 相对参考模型的对数概率比，加上一个常数。

    把 reward 代入 BT 偏好模型（$P(y_w \succ y_l) = \sigma(r_w - r_l)$），常数项因为同一 prompt 抵消，得到 DPO loss。

    核心洞察：当前模型 $\pi_\theta$ 相对参考模型 $\pi_{\text{ref}}$ 给某回答的概率变化量 $\beta \log(\pi_\theta / \pi_{\text{ref}})$，等价于一个隐含奖励。DPO 直接在这个隐含奖励上算偏好 loss，跳过了显式训 RM。论文标题《Your Language Model is Secretly a Reward Model》就是这个意思——模型本身已经隐含了一个 reward。

    完整推导：从 RLHF 目标到 DPO loss

    RLHF 的目标是带 KL 约束的 reward 最大化：

    $$\max_{\pi_\theta} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta}[r(x,y)] - \beta \, D_{\text{KL}}[\pi_\theta(\cdot|x) \| \pi_{\text{ref}}(\cdot|x)]$$

    第一项是"reward 越大越好"，第二项是"别离参考模型太远"。$\beta$ 控制偏离强度。下面 6 步推出 DPO loss。

    展开：DPO loss 的完整推导（6 步）

    目标：从 RLHF 的 KL 约束 reward 最大化目标，推出 DPO 的 loss。

    第 1 步：展开 KL 约束

    KL 散度展开成期望形式：$D_{\text{KL}}[\pi_\theta \| \pi_{\text{ref}}] = \mathbb{E}_{y \sim \pi_\theta}[\log(\pi_\theta(y|x) / \pi_{\text{ref}}(y|x))]$。代入目标：

    $$\max_{\pi_\theta} \mathbb{E}_{x, y \sim \pi_\theta}\left[ r(x,y) - \beta \log \frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)} \right]$$

    这是 RLHF 目标的等价形式——"每条采样要 reward 大，但偏离参考模型要受惩罚"。

    第 2 步：求最优 policy（变分法/泛函极值）

    对每个 $\pi_\theta(y|x)$ 求"最优"，用 Lagrangian 变分法（约束 $\sum_y \pi_\theta(y|x) = 1$）。对每个 $y$，最优解的形式是：

    $$\pi^*(y|x) \propto \pi_{\text{ref}}(y|x) \cdot \exp\left(\frac{r(x,y)}{\beta}\right)$$

    这个结果的直觉：reward $r(x,y)$ 越大，最优 $\pi^*$ 越倾向于选这个 $y$（指数项越大）；但 KL 约束让 $\pi^*$ 不能离 $\pi_{\text{ref}}$ 太远（基础是 $\pi_{\text{ref}}$）。两者的平衡就是这个指数形式。

    严格推导：对每个 $\pi_\theta(y|x)$，目标函数（去掉期望符号，逐 $y$ 看）的偏导是 $r(x,y) - \beta(\log(\pi_\theta(y|x)/\pi_{\text{ref}}(y|x)) + 1)$，令其为零，加归一化约束，得到上面的形式。

    第 3 步：归一化得到最优 policy

    概率分布要求和为 1，引入归一化常数 $Z(x)$（partition function）：

    $$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{\text{ref}}(y|x) \cdot \exp\left(\frac{r(x,y)}{\beta}\right)$$

    其中 $Z(x) = \sum_y \pi_{\text{ref}}(y|x) \cdot \exp(r(x,y)/\beta)$，只依赖 $x$，不依赖 $y$。

    第 4 步：反解——用 policy 表达 reward

    对第 3 步两边取对数：

    $$\log \pi^*(y|x) = \log \pi_{\text{ref}}(y|x) + \frac{r(x,y)}{\beta} - \log Z(x)$$

    移项解出 $r(x,y)$：

    $$r(x,y) = \beta \log \frac{\pi^*(y|x)}{\pi_{\text{ref}}(y|x)} + \beta \log Z(x)$$

    关键观察：reward 可以用 policy 相对参考模型的对数比表达。$\beta \log Z(x)$ 是一个只依赖 $x$ 的常数项。

    第 5 步：代入 BT 偏好模型

    BT 模型：$P(y_w \succ y_l | x) = \sigma(r(x,y_w) - r(x,y_l))$。代入第 4 步的 reward：

    $$r(x,y_w) - r(x,y_l) = \beta \log \frac{\pi^*(y_w|x)}{\pi_{\text{ref}}(y_w|x)} + \beta \log Z(x) - \beta \log \frac{\pi^*(y_l|x)}{\pi_{\text{ref}}(y_l|x)} - \beta \log Z(x)$$

    关键：$\beta \log Z(x)$ 在 chosen 和 rejected 上都有，因为同一个 prompt $x$。两项相减，$\beta \log Z(x)$ 抵消：

    $$r(x,y_w) - r(x,y_l) = \beta \log \frac{\pi^*(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi^*(y_l|x)}{\pi_{\text{ref}}(y_l|x)}$$

    代入 BT 模型：

    $$P(y_w \succ y_l | x) = \sigma\left( \beta \log \frac{\pi^*(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi^*(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right)$$

    第 6 步：最大似然 → DPO loss

    把 $\pi^*$ 换成可训练的 $\pi_\theta$（我们要训 $\pi_\theta$ 去逼近最优 $\pi^*$）。观测到"$y_w$ 比 $y_l$ 好"这个事实，最大似然就是让 $P$ 越大越好。等价于最小化 $-\log P$：

    $$\mathcal{L}_{\text{DPO}} = -\log \sigma\left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)} \right)$$

    这就是 DPO 的 loss 公式。形式上是一个二元分类 loss（chosen vs rejected 的偏好分类）。

    代入数值（5 token 词表演示）：

    设 prompt 是 token 0，chosen 是 token 1，rejected 是 token 2。$\beta = 0.5$。参考模型对 prompt 输出的 logits $= [-0.5, 0.8, 0.3, 0.1, -0.2]$，softmax 后：

    $$p_{\text{ref}} = [0.099, 0.364, 0.221, 0.181, 0.134]$$

    取对数：$\log \pi_{\text{ref}}(y_w) = \log 0.364 \approx -1.010$, $\log \pi_{\text{ref}}(y_l) = \log 0.221 \approx -1.510$。

    训练前：$\pi_\theta = \pi_{\text{ref}}$，对数比都是 0。$\Delta = 0.5 \cdot 0 - 0.5 \cdot 0 = 0$，$\sigma(0) = 0.5$，$L = -\log 0.5 = 0.693$（中性状态）。

    训练后（$\theta$ 更新使 logits 变为 $[-0.6, 1.3, -0.2, 0.15, -0.1]$）：softmax 后 $p_\theta = [0.077, 0.517, 0.115, 0.164, 0.127]$。

    $\log \pi_\theta(y_w) = \log 0.517 \approx -0.660$, $\log \pi_\theta(y_l) = \log 0.115 \approx -2.162$。

    对数比：$\log(\pi_\theta(y_w)/\pi_{\text{ref}}(y_w)) = -0.660 - (-1.010) = 0.350$。

    $\log(\pi_\theta(y_l)/\pi_{\text{ref}}(y_l)) = -2.162 - (-1.510) = -0.651$。

    $\Delta = 0.5 \cdot 0.350 - 0.5 \cdot (-0.651) = 0.175 + 0.326 = 0.501$。

    $\sigma(0.501) = 0.623$，$L = -\log 0.623 \approx 0.474$。

    loss 从 0.693 降到 0.474，模型学会在 prompt 下偏好 chosen。

    关键符号解释：

    • $\pi_\theta(y|x)$：当前模型在 $x$ 下生成 $y$ 的概率
    • $\pi_{\text{ref}}(y|x)$：参考模型在 $x$ 下生成 $y$ 的概率（冻结）
    • $\beta$：偏离参考模型的惩罚强度（同时是 reward 缩放因子）
    • $\sigma(x) = 1/(1 + e^{-x})$：sigmoid，把任意实数压到 $(0, 1)$
    • $Z(x)$：归一化常数，只依赖 $x$。在 BT 模型里自动抵消
    • $r(x,y)$：reward 函数（RLHF 里显式训 RM 学出来，DPO 里用 policy 隐含表达）

    来源：DPO 推导 ← Rafailov et al. 2023 §4；KL 约束 reward 最大化 + 解析最优解 ← 强化学习/控制论领域常识（soft Bellman update）；BT 模型 ← Bradley & Terry 1952。

    梯度推导：DPO 怎么训

    DPO loss 仍然走 backward。算 $\nabla_\theta \mathcal{L}_{\text{DPO}}$，发现梯度有"升 chosen + 降 rejected"的对称结构：

    $$\nabla_\theta \mathcal{L}_{\text{DPO}} = -\beta \, \sigma(-u) \left( \nabla_\theta \log \pi_\theta(y_w|x) - \nabla_\theta \log \pi_\theta(y_l|x) \right)$$

    其中 $u = \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}$ 是 DPO loss 里的 sigmoid 输入。

    展开：DPO 梯度的完整推导（5 步）

    目标：算 $\nabla_\theta \mathcal{L}_{\text{DPO}}$。

    第 1 步：简化记号

    设 $u = \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}$。$\mathcal{L} = -\log \sigma(u)$。$\pi_{\text{ref}}$ 冻结，不参与梯度。

    第 2 步：算 $d\mathcal{L}/du$

    用 sigmoid 的导数 $\sigma'(u) = \sigma(u)(1 - \sigma(u))$（标准结果）：

    $$\frac{d\mathcal{L}}{du} = -\frac{d}{du} \log \sigma(u) = -\frac{\sigma'(u)}{\sigma(u)} = -\frac{\sigma(u)(1 - \sigma(u))}{\sigma(u)} = -(1 - \sigma(u))$$

    用 sigmoid 恒等式 $1 - \sigma(u) = \sigma(-u)$：

    $$\frac{d\mathcal{L}}{du} = -\sigma(-u)$$

    第 3 步：链式法则

    $\nabla_\theta \mathcal{L} = \frac{d\mathcal{L}}{du} \cdot \nabla_\theta u = -\sigma(-u) \cdot \nabla_\theta u$

    第 4 步：展开 $\nabla_\theta u$

    $u = \beta \log \pi_\theta(y_w|x) - \beta \log \pi_{\text{ref}}(y_w|x) - \beta \log \pi_\theta(y_l|x) + \beta \log \pi_{\text{ref}}(y_l|x)$

    $\pi_{\text{ref}}$ 不依赖 $\theta$，梯度为 0：

    $$\nabla_\theta u = \beta \nabla_\theta \log \pi_\theta(y_w|x) - \beta \nabla_\theta \log \pi_\theta(y_l|x)$$

    $$= \beta \left( \nabla_\theta \log \pi_\theta(y_w|x) - \nabla_\theta \log \pi_\theta(y_l|x) \right)$$

    第 5 步：合并

    $$\nabla_\theta \mathcal{L} = -\sigma(-u) \cdot \beta \left( \nabla_\theta \log \pi_\theta(y_w|x) - \nabla_\theta \log \pi_\theta(y_l|x) \right)$$

    $$= -\beta \, \sigma(-u) \left( \nabla_\theta \log \pi_\theta(y_w|x) - \nabla_\theta \log \pi_\theta(y_l|x) \right)$$

    直觉：

    • $-\beta \, \sigma(-u)$ 是标量权重（负数，因为 $\beta > 0$, $\sigma > 0$）
    • 梯度结构是"chosen 梯度 - rejected 梯度"——同时推 chosen 上升和 rejected 下降
    • $\sigma(-u)$ 是"学习权重"：偏好越错（$u$ 越负，$\sigma(-u) \to 1$），梯度越大；偏好越对（$u$ 越正，$\sigma(-u) \to 0$），梯度越小（已经学会了，不用再调）

    和 SFT 的对比：SFT 只推正确答案上升（$\nabla \log \pi(y_{\text{正确}})$），DPO 同时推 chosen 上升和 rejected 下降，结构对称。这是 DPO 区别于 SFT 的核心。

    来源：DPO 梯度推导 ← Rafailov et al. 2023 §4.3。

    训练流程

    DPO 的训练流程极其简单，和 SFT 几乎一样：

    DPO 训练循环：

    1. 从偏好数据集采一个 batch $(x, y_w, y_l)$
    2. 用当前 $\pi_\theta$ 算 $\log \pi_\theta(y_w|x)$ 和 $\log \pi_\theta(y_l|x)$（一次前向，两个回答）
    3. 用冻结的 $\pi_{\text{ref}}$ 算 $\log \pi_{\text{ref}}(y_w|x)$ 和 $\log \pi_{\text{ref}}(y_l|x)$（只算一次缓存住）
    4. 算 DPO loss = $-\log \sigma(\beta \log \frac{\pi_\theta(y_w)}{\pi_{\text{ref}}(y_w)} - \beta \log \frac{\pi_\theta(y_l)}{\pi_{\text{ref}}(y_l)})$
    5. loss.backward() 算梯度
    6. optimizer.step() 更新 $\theta$

    对比 RLHF 的 3 步流程（训 RM → 用 RM 当环境 → 用 PPO 训 policy），DPO 一个 loss 搞定，工程上简单很多。

    三、总结：DPO vs RLHF

    本章你将知道：DPO 和 RLHF 的对比，什么时候用哪个。

    维度RLHFDPO
    步数3 步（数据 → RM → PPO）1 步（直接在偏好数据上算 loss）
    需要训 RM是否（reward 用 policy 隐含表达）
    需要 RL 算法是（PPO）否（就是普通的 backward + SGD/Adam）
    能持续探索是（policy 可以采样新回答让 RM 打分）否（只用静态偏好数据，不能发现新回答）
    训练稳定性较差（PPO 超参难调，reward hacking）好（和 SFT 一样稳）
    效果上限较高（持续探索能找到更好的回答）受限于偏好数据覆盖范围
    代表模型ChatGPT、ClaudeLlama 3、Qwen

    工业界现在流行 DPO 做 base，RLHF 做精细调整。两者不是对立的，可以组合用：先 DPO 跑出基础对齐（便宜稳定），再用 RLHF 持续探索（贵但效果好）。

    什么时候用 DPO：偏好数据充足、追求训练稳定、不想引入 RL 复杂度（大多数开源模型）。

    什么时候用 RLHF：需要持续探索新回答、有预算训 RM 和调 PPO 超参（大型闭源模型）。
