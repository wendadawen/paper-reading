    GRPO：用组内相对奖励省掉 value model

    问题：PPO 的 advantage 要 value model 当 baseline，标准做法里 value model 和 policy 一样大，显存翻倍还难训
    方案：对同一 prompt 采样一组回答，用组内 reward 的均值当 baseline，省掉 value model
    产物：一个能稳定训推理模型（DeepSeek-R1 类）的轻量 RL 算法

    来源：DeepSeekMath 论文（Shao et al. 2024）§3；DeepSeek-R1（DeepSeek-AI 2025）§2.2

    前置：PPO（importance ratio + clip）、reward、advantage、baseline。不会的先看 RLHF。

    GRPO 一句话：PPO 要训一个 value model 来估 baseline，标准做法里它通常和 policy 同样大（InstructGPT、DeepSeekMath 都这么做）。GRPO 把它换成"对同一 prompt 采一组回答，用组内 reward 的均值当 baseline"。这样省一半显存，且对"reward 稀疏明确"的推理任务（数学、代码）更稳定。DeepSeek-R1 用的就是它。

    一、问题：PPO 要训 value model，太贵且不稳

    本章你将知道：PPO 的 advantage 怎么来，value model 为啥是负担。

    PPO 用 advantage $A = G - V(s)$ 训练，其中 $V(s)$ 是 value model 估出来的"这个状态的预期 return"，作为 baseline 减掉来降低梯度方差。$G$ 是这次回答实际拿到的 return（单步场景就等于 reward $r$）。

    value model 看起来人畜无害，但在 LLM 训练里有两个痛点：

    痛点 1：太贵

    标准做法里 value model 和 policy 一样大（InstructGPT 从 SFT 模型初始化 value；DeepSeekMath 论文 §3 也说"value function needs to be as large as the policy model to obtain a good estimation of value"——小了估不准）。同样的 transformer 架构、同样的参数量，这意味着：

    • 显存翻倍（多存一份模型权重）
    • 优化器状态翻倍（Adam 要给每个参数存 momentum + variance）
    • 梯度同步翻倍（多卡训练时多一份通信）

    训一个 7B policy 实际要承担 14B 的开销。训 671B 的 DeepSeek-R1？直接扛不住。

    不是数学要求：理论上可以用更小的 value model，代价是 advantage 估计更噪、训练更不稳。所以实践里为了稳，就跟着 policy 大小走。

    痛点 2：难训

    推理任务（数学、代码）里 reward 是稀疏的 0/1——只有最后才知道对错。要训 value model 预测"这个写到一半的回答最终会得多少分"，本质是个困难的回归问题。value model 估不准 → advantage 噪声大 → 训练不稳甚至崩。

    关键观察：对同一个 prompt，如果采样多个回答，它们的 reward 自然就提供了"该 prompt 的预期 return"信息——无须再用 value model 估。这就是 GRPO 的出发点。

    二、方法：用组内相对奖励当 baseline，省掉 value model

    本章你将知道：GRPO 怎么用"组内 reward"算 advantage，怎么套进 PPO 的 clip 框架。

    直觉

    PPO 用 value model 估 baseline。GRPO 换个思路：对同一 prompt 采样 $G$ 个回答，每个回答算 reward，用这 $G$ 个 reward 的均值当 baseline。某个回答的 reward 高于均值 → advantage 正 → 鼓励；低于均值 → advantage 负 → 抑制。无须 value model。

    整个方法分 5 步：采样一组 → 算 reward → 组内归一化算 advantage → 套 PPO clip → 加 KL 约束。下面按顺序讲。

    关键假设：reward 的"绝对值"无意义，"相对差异"才有意义。

    同一 prompt 下 reward = 0.8 和 0.6 的差距，跟 100 和 98 的差距，本质一样——都是"前者比后者高 0.2 个相对量"。这和 RLHF 里"RM 学的是相对排序"一脉相承，正是组内归一化的前提。

    第 1 步：对同一 prompt 采样一组回答

    对 prompt $q$，用当前策略 $\pi_{\theta_{\text{old}}}$ 采样一组回答 $\{o_1, o_2, \ldots, o_G\}$。$G$ 通常 4~64。这一组回答共享同一个 prompt，可以并行采样，开销摊销。

    第 2 步：算每个回答的 reward

    reward 来源有两种：

    • RLVR 场景：reward $= 1$（答对）/ $0$（答错）。规则验证，可靠且便宜
    • RLHF 场景：reward $= r_\varphi(q, o_i)$，RM 打分

    本文用 RLVR 举例（GRPO 的主战场），reward 是 0/1。

    第 3 步：组内归一化算 advantage

    这是 GRPO 的核心。用组内 reward 的均值和标准差归一化：

    $$A_i = \frac{r_i - \text{mean}(\{r_1, \ldots, r_G\})}{\text{std}(\{r_1, \ldots, r_G\})}$$

    展开：组内 advantage 公式完整推导

    目标：在不用 value model 的前提下，给每个回答算一个 advantage——正值表示比平均好，负值表示比平均差。

    第 1 步：从 PPO 的 advantage 出发

    PPO 的 advantage 是 $A = G - V(s)$，$V(s)$ 是 value model 估的"这个状态的预期 return"。我们要换掉 $V(s)$。

    第 2 步：用组内均值当 baseline

    对同一 prompt $q$ 采了 $G$ 个回答，每个回答有 reward $r_i$。这 $G$ 个 reward 的均值是：

    $$\bar{r} = \frac{1}{G} \sum_{j=1}^{G} r_j$$

    $\bar{r}$ 就是"该 prompt 的预期 return"的蒙特卡洛估计——无须 value model。代进 advantage：

    $$A_i = r_i - \bar{r}$$

    这一步已经能用了：$r_i > \bar{r}$ 时 $A_i > 0$ 鼓励，$r_i < \bar{r}$ 时 $A_i < 0$ 抑制。但还有个问题——$A_i$ 的绝对大小依赖 reward 的尺度。如果 reward 是 0/1，$A_i \in [-0.5, 0.5]$；如果 reward 是 0/100，$A_i \in [-50, 50]$。尺度不一样，学习率就要跟着调，很烦。

    第 3 步：除以标准差做归一化

    用组内标准差归一化：

    $$\text{std}(\{r_1, \ldots, r_G\}) = \sqrt{\frac{1}{G} \sum_{j=1}^{G} (r_j - \bar{r})^2}$$

    advantage 变成 Z-score 形式：

    $$A_i = \frac{r_i - \bar{r}}{\text{std}(\{r_1, \ldots, r_G\})}$$

    这样 $A_i$ 的尺度固定（大致在 $[-\sqrt{G}, \sqrt{G}]$ 范围），与 reward 的绝对大小无关。学习率不用跟着 reward 尺度调。

    第 4 步：边缘情况——std = 0 怎么办

    如果 $G$ 个回答 reward 全一样（全对或全错），$\text{std} = 0$，分母为 0。这时所有 $A_i = 0$，梯度消失，相当于"这一组没学到东西，跳过"。这其实是正确行为：全对说明太简单（无须学），全错说明太难（学不动）。代码实现里分母加个小 $\epsilon$ 防除零。

    关键符号解释：

    • $q$：prompt
    • $o_i$：第 $i$ 个采样回答，$i \in \{1, \ldots, G\}$
    • $r_i$：第 $i$ 个回答的 reward
    • $\bar{r} = \text{mean}(\{r_1, \ldots, r_G\})$：组内均值，当 baseline
    • $\text{std}(\{r_1, \ldots, r_G\})$：组内标准差，用于归一化
    • $A_i$：第 $i$ 个回答的 advantage

    来源：Shao et al. 2024 (DeepSeekMath) §3.2，公式 (5)。

    第 4 步：套进 PPO clip 目标

    有了 advantage，套 PPO 的 importance ratio + clip 框架（这两个概念在 RLHF 文档有完整推导，这里直接用）。

    定义 importance ratio：

    $$\rho_i(\theta) = \frac{\pi_\theta(o_i|q)}{\pi_{\theta_{\text{old}}}(o_i|q)}$$

    $\rho_i = 1$ 表示新旧策略一样；$\rho_i > 1$ 表示新策略更偏好这个回答；$\rho_i < 1$ 反之。

    套 PPO clip（取 min 做悲观估计，超出 $[1-\epsilon, 1+\epsilon]$ 梯度消失）：

    $$\min\left( \rho_i(\theta) A_i, \, \text{clip}(\rho_i(\theta), 1-\epsilon, 1+\epsilon) A_i \right)$$

    展开：为什么 clip 取 min 是"自动刹车"

    目标：理解 PPO clip 的保护机制——为什么超出 $[1-\epsilon, 1+\epsilon]$ 梯度会消失。

    分两种情况：

    情况 1：$A_i > 0$（好回答，要鼓励 $\rho_i$ 增大）

    不 clip 时项是 $\rho_i A_i$，$\rho_i$ 越大越好。但 clip 把 $\rho_i$ 封顶在 $1+\epsilon$：

    • $\rho_i \leq 1+\epsilon$：clip 不动，取 min 选 $\rho_i A_i$，梯度正常鼓励 $\rho_i$ 上升
    • $\rho_i > 1+\epsilon$：clip 取 $1+\epsilon$，$\text{clip} \cdot A_i = (1+\epsilon) A_i < \rho_i A_i$，取 min 选前者，梯度消失——停止鼓励，防止 policy 过度偏好这个回答

    情况 2：$A_i < 0$（差回答，要鼓励 $\rho_i$ 减小）

    不 clip 时项是 $\rho_i A_i$（注意 $A_i$ 是负的），$\rho_i$ 越小越好（项越大）。但 clip 把 $\rho_i$ 托底在 $1-\epsilon$：

    • $\rho_i \geq 1-\epsilon$：clip 不动，取 min 选 $\rho_i A_i$，梯度正常鼓励 $\rho_i$ 下降
    • $\rho_i < 1-\epsilon$：clip 取 $1-\epsilon$，$\text{clip} \cdot A_i = (1-\epsilon) A_i > \rho_i A_i$（$A_i$ 负，$(1-\epsilon)A_i$ 比 $\rho_i A_i$ 大），取 min 选 $\rho_i A_i$，梯度消失——停止惩罚，防止 policy 过度回避

    两边都封住：$\rho_i \in [1-\epsilon, 1+\epsilon]$ 梯度正常，超出范围梯度消失。这就是 clip 的"自动刹车"——允许 policy 变好，但不能一步跳太远。$\epsilon$ 通常取 0.1~0.3。

    来源：Schulman et al. 2017 (PPO 论文)。

    第 5 步：加 KL 散度约束

    纯 PPO 目标可能让 $\pi_\theta$ 漂离参考模型 $\pi_{\text{ref}}$（通常是 SFT 后的模型）太远，导致：

    • reward hacking：policy 钻 reward 函数的空子，生成奇怪的 token 序列骗高分
    • 灾难性遗忘：忘记 SFT 阶段学到的语言流畅性

    GRPO 加一个 KL 散度惩罚项约束漂移：

    $$- \beta \, D_{KL}\big(\pi_\theta \,\|\, \pi_{\text{ref}}\big)$$

    $\beta$ 是惩罚系数（DeepSeek 实现里默认 0.01 左右）。$\beta$ 大 → 更紧贴 $\pi_{\text{ref}}$；$\beta$ 小 → 更激进探索。

    展开：GRPO 用的 KL 估计器公式

    目标：在 token 级别估 $\pi_\theta$ 和 $\pi_{\text{ref}}$ 的 KL 散度，且要无偏、低方差。

    第 1 步：KL 散度的定义

    两个分布 $p$ 和 $q$ 的 KL 散度：

    $$D_{KL}(p \| q) = \mathbb{E}_{x \sim p}\left[\log \frac{p(x)}{q(x)}\right]$$

    直接算要遍历词表求期望，开销大。要用采样估计。

    第 2 步：朴素的采样估计

    从 $\pi_\theta$ 采 $o$，用 $\log \frac{\pi_\theta(o|q)}{\pi_{\text{ref}}(o|q)}$ 估 KL。但这个估计方差大（小概率 token 贡献大），数值不稳。

    第 3 步：Schulman 的 k3 估计器

    John Schulman 提过一个低方差无偏估计器（GRPO 论文用的就是这个）：

    $$D_{KL}\big(\pi_\theta(\cdot|q) \,\|\, \pi_{\text{ref}}(\cdot|q)\big) \approx \frac{\pi_{\text{ref}}(o|q)}{\pi_\theta(o|q)} - \log \frac{\pi_{\text{ref}}(o|q)}{\pi_\theta(o|q)}$$

    令 $r = \pi_{\text{ref}}(o|q) / \pi_\theta(o|q)$，则估计器为 $r - \log r$。这个形式在 $r = 1$（两分布相同）时为 0，且对偏离 1 的两边都给非负的惩罚，方差小。

    实现时按 token 级别算，对一条回答的所有 token 求和/求平均。

    关键符号解释：

    • $\pi_\theta(\cdot|q)$：当前策略对 prompt $q$ 的下一个 token 分布
    • $\pi_{\text{ref}}(\cdot|q)$：参考模型对同一 prompt 的分布
    • $o$：从 $\pi_\theta$ 采样的 token
    • $r = \pi_{\text{ref}}(o|q) / \pi_\theta(o|q)$：参考模型对采样 token 的相对概率

    来源：Schulman 2016 (blog "Approximating KL Divergence")；DeepSeekMath §3.2 实现采用。

    完整 GRPO 目标

    把第 3~5 步合起来，对一个 prompt $q$ 的 $G$ 个采样取平均，再对所有 prompt 求期望：

    $$J_{\text{GRPO}}(\theta) = \mathbb{E}_{q, \{o_i\} \sim \pi_{\theta_{\text{old}}}} \left[ \frac{1}{G} \sum_{i=1}^{G} \left( \min\big(\rho_i A_i, \, \text{clip}(\rho_i, 1-\epsilon, 1+\epsilon) A_i\big) - \beta \, D_{KL}\big(\pi_\theta \,\|\, \pi_{\text{ref}}\big) \right) \right]$$

    训练时最大化 $J_{\text{GRPO}}$（等价于最小化 $-J_{\text{GRPO}}$）。每个 batch 跑多个 prompt，每个 prompt 采 $G$ 个回答，组内算 advantage，套 clip + KL，反传更新 $\theta$。

    数值例子：用 GRPO 训 "1+1="

    prompt $q =$ "1+1="，采 $G = 4$ 个回答（设 policy 是 SFT 后但还没训过加法的模型，答对答错都有）：

    采样 $i$回答 $o_i$reward $r_i$（对错验证）
    1"2"1.0
    2"2"1.0
    3"3"0.0
    4"3"0.0
    展开：从 reward 到 advantage 的完整计算

    第 1 步：算组内均值

    $$\bar{r} = \frac{1.0 + 1.0 + 0.0 + 0.0}{4} = 0.5$$

    $\bar{r} = 0.5$ 就是"这个 prompt 的预期 return"——baseline。

    第 2 步：算组内标准差

    $$\text{std} = \sqrt{\frac{(1.0-0.5)^2 + (1.0-0.5)^2 + (0.0-0.5)^2 + (0.0-0.5)^2}{4}} = \sqrt{\frac{0.25 \times 4}{4}} = \sqrt{0.25} = 0.5$$

    第 3 步：算每个回答的 advantage

    $$A_1 = \frac{1.0 - 0.5}{0.5} = +1.0 \quad (\text{答对，鼓励})$$

    $$A_2 = \frac{1.0 - 0.5}{0.5} = +1.0 \quad (\text{答对，鼓励})$$

    $$A_3 = \frac{0.0 - 0.5}{0.5} = -1.0 \quad (\text{答错，抑制})$$

    $$A_4 = \frac{0.0 - 0.5}{0.5} = -1.0 \quad (\text{答错，抑制})$$

    第 4 步：看效果

    advantage 套进 clip 目标后，梯度推 $\pi_\theta(\text{"2"} | \text{"1+1="})$ 上升、$\pi_\theta(\text{"3"} | \text{"1+1="})$ 下降。无须 value model——只用 4 个采样就算出了 baseline。

    对比 PPO 的做法：PPO 要先训一个 value model，让它学到 $V(\text{"1+1="}) \approx 0.5$，再算 $A_i = r_i - 0.5$。GRPO 直接从采样里把 $0.5$ 算出来，省掉 value model 的训练成本。

    自适应难度：组内归一化自带"难度自适应"。

    • 难题（多数答错，$\bar{r}$ 低）：只要有一个答对，advantage 就是大的正值，大力鼓励
    • 简单题（多数答对，$\text{std}$ 小）：advantage 接近 0，梯度小，自动跳过，把训练算力留给难题

    这是 GRPO 在数学/代码任务上的隐藏好处——value model 学不出这种自适应，得人为调课程。

    三、总结：GRPO 把 value model 换成组内 baseline

    本章你将知道：方法本质回顾 + 和 PPO 的对比。

    GRPO 的核心改动：把 PPO 的 $V(s)$ baseline 换成组内 $\bar{r}$ baseline。一个数学等价替换，省掉一个和 policy 一样大的模型，且对 reward 稀疏的推理任务更稳。其他部分（importance ratio、clip、KL 约束）和 PPO 完全一样。

    维度PPOGRPO
    value model需要不需要
    baseline$V(s)$（学出来）$\bar{r}$（算出来）
    advantage$A = G - V(s)$$A_i = (r_i - \bar{r}) / \text{std}$
    每 prompt 采样数1$G$（4~64）
    显存（同 policy 大小）2×（policy + value）1×（policy）
    适合场景通用 RLHF（开放对话）reward 明确的推理（数学/代码）
    代表模型ChatGPT、ClaudeDeepSeek-R1、Qwen2-Math

    注意 GRPO 不是免费午餐：每个 prompt 要采 $G$ 个回答，采样开销比 PPO 大 $G$ 倍。换来的是省掉 value model 的训练和显存。在 reward 明确、采样可并行的推理任务上这笔交易很划算；在 reward 模糊、采样昂贵的开放对话上未必。

    工业界推理模型（o1 / DeepSeek-R1 类）主流用 GRPO + RLVR：reward 来自答案对错验证，GRPO 省掉 value model，把训练算力集中在 policy 上。
